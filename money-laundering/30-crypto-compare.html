合规操作指南：加密资产币种级风控策略

VASP/CEX风控团队的币种级准入与监控标准

不同加密币种的风险特征差异极大。合规团队应为每种交易币种设定明确的风控参数和监控规则。

  1. 币种风险分级：按匿名性/流动性/可追踪性三维评分将币种分为红(禁止交易)/黄(强化监控)/绿(标准监控)三色。BTC/ETH → 黄(需链上标签+风险评分)；XMR/ZEC/GRIN → 红(禁止充提/交易)；USDC → 绿(强合规发行人可快速审批)；USDT → 黄(可冻结但流动性最好，需重点监控)。
  2. 充值地址预筛查：客户申请充提某一币种时，自动查询该地址的历史交易路径(最多10跳)。若发现与混币器/暗网/制裁地址关联 → 禁止充值或要求二级KYC(视频面谈+地址用途声明)。
  3. 交易监控规则差异化：对不同币种配置不同的监控阈值。(a) BTC/ETH：单笔≥$50k自动风险标签查询；(b) USDT(TRC20)：单笔≥$30k触发；(c) USDC：单笔≥$100k触发(因其强合规属性)；(d) SOL/DOT：单笔≥$50k触发。
  4. 提现审批流程：大额提现(≥客户风险等级限额)必须执行(1)链上目的地地址风险标签；(2)是否为CEX/托管钱包/非托管钱包判断；(3)若目的地为CEX → 检查该CEX是否在FATF Travel Rule合规名单上。

实操教程：各币种链上监控SOP（以Chainalysis/TRM Labs为例）

VASP合规团队必知的币种级链上监控落地步骤

以下SOP适用于在Chainalysis Radar或TRM Labs平台上为不同加密资产配置监控规则。每一步都对应一个具体的后台操作。

▌SOP：Chainalysis Radar币种类监控规则配置

  1. 添加目标币种地址监控（第1-10步）：(1) 登录Chainalysis Compliance Suite → 点击"Monitoring"；(2) 点击"Add Watch Address"；(3) 输入客户充币地址（支持批量CSV导入）；(4) 为该地址选择币种类别（BTC/ETH/USDT-ERC20/USDT-TRC20等）；(5) 设置风险级别（基于KYC完成度自动判定Low/Medium/High）；(6) 配置通知条件（Incoming/Outgoing/Both）；(7) 设置单笔金额阈值（如≥$50,000则触发预警）；(8) 启用"标签检查"功能（自动匹配Chainalysis Risk Labels）；(9) 设置最大跳跃深度（Depth = 5，即追踪5跳以内的对手方地址）；(10) 勾选"自动生成Risk Score每日报告"。
  2. 配置混币器/暗网监控规则（第11-18步）：(11) 在"Entities"模块添加黑名单实体：Tornado Cash、ChipMixer、BigWaterMix、DeepMix、DarkMix、Known Crypto-Scammer、Known Crypto-Scammer、Stolen Hack Funds（Elliptic Database导入）；(12) 进入"Rules Engine" → "New Rule" → 选择"Counterparty Risk"类型；(13) 设定条件：if any transaction counterparty (within ≤3 hops) matches blacklisted entity → severity = Critical；(14) 同时设定：if transaction flow pattern resembles mixing (≥3 addresses branching from same source, each sending to ≥2 destinations) → severity = High；(15) 关联自动化响应：标记待审核(Flag for Review)+发送MLRO通知；(16) 对高风险币种(XMR/ZEC)客户额外添加"地址来源调查"步骤：要求客户提供资金来源证明；(17) 设置月度自动审计——导出所有"已关闭"告警，抽查5%确认是否漏报；(18) 每季度更新一次黑名单位址库（Chainalysis每月更新、Elliptic每周更新、OFAC SDN名单每周更新）。
  3. USDT vs BTC 差异化监控配置：(19) USDT(ERC20/TRC20)单独配置规则：因Tether可远程冻结地址，需额外监控"大额集中转入→分散转出"模式（典型USDT洗钱特征）；(20) 对USDT设置"集中指数"规则——若单个地址在24小时内收到来自≥20个不同地址的资金然后转给≥30个地址，触发"聚合器检查"；(21) BTC监控重点在于CoinJoin检测：检测同一输入中有≥5个其他无关输入（CoinJoin的典型环大小），使用Chainalysis的"CoinJoin Detection"插件；(22) USDC/USDE等强合规稳定币配置简化规则——因为发行人有黑名单能力，洗钱者使用较少，监控频率可适当降低。
  4. 跨链资产监控设置：(23) 启用跨链桥接监控：当客户资产从一条链转移到另一条链时（如ETH→BSC via Aave Bridge），自动创建关联案例(Case)；(24) 对SOL/DOT/AVAX等L1链配置独立的地址监控组；(25) 配置跨链"跳跃分析"：如果一个客户同时使用BSC和Polygon，系统应自动识别这是同一实体的"跨链套利"模式还是两个不同用户在使用相同策略。
