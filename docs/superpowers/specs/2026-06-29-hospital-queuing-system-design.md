# 医院门诊叫号分诊系统设计方案

> **日期**: 2026-06-29
> **项目**: 二级医院门诊叫号分诊系统整体解决方案
> **状态**: 待审批

---

## 一、项目背景

某二级医院拟在各科室主治医生诊室部署叫号机与分诊屏系统，实现患者智能排队、自动叫号、分诊屏信息展示等功能。医院规模为5-10个科室，日均门诊量100-500人，医生5-30人。

## 二、需求汇总

| 维度 | 需求描述 |
|------|----------|
| 医院规模 | 二级医院，5-10科室，100-500人/天，5-30医生 |
| 叫号规则 | 组合规则：普通顺序 + 急诊插队 + 按科室独立排队 |
| 取号方式 | 线上预约 + 到院取号（微信/小程序） |
| 分诊屏内容 | 叫号信息 + 医生介绍 + 医院公告 + 多媒体宣传 |
| HIS对接 | 部分对接，仅挂号数据实时同步 |

## 三、系统总体架构

```
┌─────────────────────────────────────────────────────┐
│                    患者端                             │
│         微信小程序 / 公众号 / APP                      │
│     (预约 → 取号 → 等待推送 → 评价 → 复诊提醒)          │
└──────────────────────┬──────────────────────────────┘
                       │ HTTPS/JSON
┌──────────────────────▼──────────────────────────────┐
│                  应用层服务                            │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌─────────┐ │
│  │ 排队管理  │ │ 叫号引擎  │ │ 消息推送  │ │ 报表统计 │ │
│  └──────────┘ └──────────┘ └──────────┘ └─────────┘ │
└──────────────────────┬──────────────────────────────┘
                       │ RESTful API / HL7
┌──────────────────────▼──────────────────────────────┐
│                  数据层                               │
│  ┌────────────┐  ┌────────────┐  ┌───────────────┐  │
│  │  MySQL主库  │  │  Redis缓存  │  │  挂号数据中间库 │  │
│  └────────────┘  └────────────┘  └───────────────┘  │
└───────┬───────────────────────┬─────────────────────┘
        │                       │
┌───────▼────────┐    ┌────────▼──────────────────────┐
│   HIS系统(HIS)  │    │       硬件终端层               │
│ (挂号数据同步)  │    │  叫号机×10  分诊屏×10  音响×10 │
└────────────────┘    │  医生控制器×10  网络设备         │
                      └───────────────────────────────┘
```

## 四、三套方案对比

### 方案A：经济入门版（11万元）

| 项目 | 说明 |
|------|------|
| 定位 | 最低成本快速上线，满足基本功能 |
| 架构 | 单机部署，本地数据库，无实时HIS对接 |
| 软件 | 独立服务端 + 微信小程序 + 手动/批量数据导入 |
| 硬件 | 10台10寸触摸屏叫号机 + 10台55寸分诊屏 + 基础网络 |
| HIS对接 | 无（每日手动导入或CSV批量导入） |
| 实施周期 | 1-2周 |
| 年维护费 | ~1万元 |
| 总价 | **110,000元** |

## 五、核心子系统拆解（以方案B为例）

### 5.1 叫号引擎子系统

| 功能模块 | 说明 |
|----------|------|
| 排队管理 | 按科室独立队列，支持急诊插队、VIP优先 |
| 叫号逻辑 | 语音播报 + 屏幕显示 + 短信/微信推送 |
| 规则引擎 | 可配置叫号间隔、超时重呼、跳过机制 |
| 异常处理 | 患者未到号自动跳过、医生暂停接诊 |

### 5.2 分诊显示子系统

| 功能模块 | 说明 |
|----------|------|
| 叫号信息展示 | 当前号码、等待列表、预计等待时间 |
| 医生信息卡片 | 头像、职称、擅长领域、出诊时间 |
| 公告轮播 | 医院通知、健康科普、科室介绍 |
| 多媒体播放 | 宣传片、健康教育视频（定时开关） |

### 5.3 患者服务子系统

| 功能模块 | 说明 |
|----------|------|
| 在线预约 | 微信/小程序预约挂号，选择时段 |
| 到院取号 | 扫码或输入验证码取号 |
| 实时推送 | 排队进度微信推送，剩余3人时提醒 |
| 诊后评价 | 满意度评分、意见留言 |

### 5.4 HIS数据对接子系统

| 对接内容 | 方式 | 频率 |
|----------|------|------|
| 挂号数据 | 中间库读取 / HL7消息 | 实时 |
| 医生排班 | 中间库读取 | 实时 |
| 就诊状态 | 医生端回写 | 实时 |
| 统计数据 | 每日定时同步 | 每日 |

### 5.5 管理后台子系统

| 功能模块 | 说明 |
|----------|------|
| 科室管理 | 科室增删改、诊室分配 |
| 医生管理 | 医生信息、排班设置 |
| 规则配置 | 叫号优先级、超时时间、重呼次数 |
| 内容管理 | 公告发布、视频上传、屏保设置 |
| 数据统计 | 就诊量、等待时长、医生绩效报表 |

## 十、风险与合规

| 风险类别 | 说明 | 应对措施 |
|----------|------|----------|
| 数据隐私 | 患者信息涉及PIPL合规 | 数据脱敏展示、加密传输、权限管控 |
| 网络安全 | 医院信息系统等保要求 | 内网隔离、防火墙、日志审计 |
| 系统可用性 | 叫号中断影响就诊秩序 | 双机热备、离线模式降级 |
| 硬件故障 | 屏幕/叫号机损坏 | 备件储备、4小时响应维保 |
| HIS对接 | 接口不稳定导致数据丢失 | 中间库缓冲、断点续传、人工兜底 |
