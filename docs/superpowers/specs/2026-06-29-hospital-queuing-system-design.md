# 医院门诊叫号分诊系统设计方案

> **日期**: 2026-06-29
> **项目**: 二级医院门诊叫号分诊系统整体解决方案
> **状态**: 待审批

---

## 一、项目背景

某二级医院拟在各科室主治医生诊室部署叫号机与分诊屏系统，实现患者智能排队、自动叫号、分诊屏信息展示等功能。医院规模为5-10个科室，日均门诊量100-500人，医生5-30人。

## 二、需求汇总

| 维度 | 需求描述 |
|------|----------|
| 医院规模 | 二级医院，5-10科室，100-500人/天，5-30医生 |
| 叫号规则 | 组合规则：普通顺序 + 急诊插队 + 按科室独立排队 |
| 取号方式 | 线上预约 + 到院取号（微信/小程序） |
| 分诊屏内容 | 叫号信息 + 医生介绍 + 医院公告 + 多媒体宣传 |
| HIS对接 | 部分对接，仅挂号数据实时同步 |

## 十、最终推荐建议

### 强烈推荐方案B（标准商务版）

**推荐理由：**

1. **性价比最优**: 34.8万首次投资，5年TCO仅42.3万，远低于方案C的166万
2. **二级医院完美匹配**: 5-10个科室规模，方案B功能完全覆盖且无浪费
3. **实施风险最低**: 部分对接HIS降低复杂度，同时保证数据实时性
4. **可扩展路径清晰**: 未来可直接平滑升级到方案C架构，无需推倒重来
5. **市场验证充分**: 同类方案在全国二级医院已有数百成功案例

### 备选建议

- 如果预算非常紧张（<15万），选**方案A**先跑起来，后续再升级
- 如果医院有分院扩张计划或追求品牌形象，直接上**方案C**一步到位
