第二十四章：云平台 — 数据存储/HIPAA合规/API设计/弹性扩展

云平台的可靠性直接决定了设备在线时的数据一致性和紧急呼叫的响应速度。本章节设计基于阿里云的混合云架构，满足中国数据安全法规及可能的出海需求。

物联网平台配置步骤 
# 1. 在产品下创建Topic类
# 主题命名规范: /sys/{productKey}/{deviceName}/thing/event/property/post
# 示例: /sys/a1XyZ.../elder_bangle_001/thing/event/heartPost

# 2. 批量创建设备(用Python脚本)
import paho.mqtt.client as mqtt

PRODUCT_KEY = "a1XyZAbCdEf"
DEVICE_NAME = "elder_bangle_001"
CLIENT_ID = "esp32|securemode=3,signmethod=hmacsha1,timestamp=1234567890|"

# 计算签名
def get_sign(device_secret, timestamp):
    import hashlib, hmac
    msg = f"{CLIENT_ID}{timestamp}"
    return hmac.new(device_secret.encode(), msg.encode(), hashlib.sha1).hexdigest()

# 3. MQTT 连接参数
client = mqtt.Client(client_id=CLIENT_ID)
client.username_pw_set(DEVICE_NAME, sign)
client.tls_set()  # TLS 端口: 1883(TCP) / 8883(TLS)

# 4. 上报数据示例
payload = json.dumps({
    "id": 12345,
    "version": "1.0",
    "params": {
        "HeartRate": {"Value": 72},
        "BloodPressure": {"Systolic": 120, "Diastolic": 80}
    }
})
client.publish("/topic/health/upload", payload, qos=1)
print("数据上报成功!")

步骤四：API Gateway 路由配置(Nginx)

nginx.conf 生产配置 
upstream api_backend {
    server 172.18.0.3:3000;
    server 172.18.0.4:3000;
    keepalive 32;
}

server {
    listen 80;
    server_name api.eldercare.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.eldercare.com;

    ssl_certificate /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;

    # 限速: 每IP每秒5请求
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=5r/s;

    location /api/v1/ {
        limit_req zone=api_limit burst=10 nodelay;
        proxy_pass http://api_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_connect_timeout 5s;
        proxy_read_timeout 30s;
    }

    # 健康数据上报端点(允许更高QPS)
    location /api/v1/device/telemetry {
        limit_req zone=api_limit burst=50 nodelay;
        proxy_pass http://api_backend/api/v1/device/telemetry;
    }

    # 告警端点(最低延迟)
    location /api/v1/alerts/trigger {
        proxy_read_timeout 3s;
        proxy_pass http://api_backend/api/v1/alerts/trigger;
    }
}
