第九章：8个月上市实施路线图 — 手环/药盒/APP并行推进

手环开发16-24周(4-6个月)，药盒16-22周(4-5.5个月)，APP MVP 3-4个月。三者并行推进，8个月实现双产品同步上市。

9.1 手环ODM开发全流程

阶段时间关键任务交付物深度解读
Phase 12周需求评审：功能清单/定位精度/续航目标/IP67防水/SIM卡方案PRD文档需求评审是"不可逆决策"的起点——一旦开模就无法热更新。建议PRD冻结后任何变更走CR(Change Request)流程，避免范围蔓延(Scope Creep)。
Phase 22-3周ODM方案选型：润科/杰理现有方案，评估MCU+定位+传感器+屏幕方案选型报告选方案本质是"技术债务"决策——杰理便宜但技术债务高(固件bug多)，润科贵但稳定。建议选润科，节省后期调试时间约2-3周。
Phase 33-4周ID/MD设计：适老化设计(大字体/大图标/实体SOS/硅胶表带/防拆卸表扣)3D渲染图+结构图ID(工业设计)决定"好不好看"，MD(机械设计)决定"好不好用"。老人对"大按钮"的需求远超年轻人预期——按钮尺寸应≥15mm直径。
Phase 43-4周开模+手板验证：3D打印→防水测试→跌落测试(1米)功能手板x10+3D打印手板只能验证外观和基本功能，不能验证量产一致性。建议同时做"软模"(铝模)手板约¥5,000-10,000，更接近量产品质。
Phase 52-3周EVT试产：工程验证测试(功能/射频/电池续航)EVT报告EVT是第一个"可量产"节点。目标：10台样机全部通过自测用例。如果通过率<80%，说明设计有系统性问题，需回退到DVT前重新设计。
Phase 62-3周DVT设计验证：高低温(-10~50°C)/盐雾/老化测试DVT报告DVT是最关键的可靠性节点。北方冬季(-10°C)锂电容量衰减40%是常见坑，需在算法层做低温补偿。盐雾测试针对沿海城市用户。
Phase 72周PVT小批量 → MP量产量产出货PVT良率目标≥95%。低于此值说明产线工艺不稳定，不建议直接转MP。建议PVT产100台，良率≥95%再启动MP。

9.2 药盒ODM开发全流程

阶段时间关键任务交付物深度解读
Phase 12周需求评审：药盒容量/分药机构方案/WiFi+BLE/续航目标PRD文档药盒PRD核心矛盾：容量vs体积。老人一次吃5-8粒药，但药盒不能太大(老人手部力量弱)。建议设计"可拆卸药仓"——每次取出一格用药，用完放回。
Phase 22-3周ODM方案选型：ESP32-C3 + 步进电机分药 + 红外/光电检测方案选型报告ESP32-C3是WiFi+BLE二合一SoC，比单独WiFi+BLE方案省¥5-8 BOM。但WiFi在老人家庭中可能不稳定(老旧路由器)，需同时支持蓝牙直连手机。
Phase 33-4周结构设计：食品级塑料药仓/防儿童锁/语音喇叭/LED指示3D结构图防儿童锁是合规要求——GB 15241-2019要求药品包装具有"儿童难以打开但老人可操作"的设计。建议用"按压+旋转"双动作解锁。
Phase 43-4周开模+手板：3D打印验证分药机构可靠性(目标>10,000次循环)功能手板x10+10,000次循环≈每天4次×约7年。实际寿命目标应≥20,000次(每天4次×约14年)，但3D打印验证成本限制在此。量产件需做加速老化测试。
Phase 54-6周EVT/DVT/PVT：可靠性验证+小批量量产出货药盒比手环多一个"机械可靠性"维度——步进电机寿命、药仓磨损、齿轮间隙。建议EVT阶段就做"连续出药1,000次无故障"测试。

细微信号：8个月窗口期正在被压缩至6个月
随着ODM公模方案成熟和微信生态IoT能力增强，行业平均上市周期已从2022年的12个月压缩至2026年的6-8个月。小米/华为等巨头从立项到上市最快仅需4个月(利用现有公模+APP模板)。我们的8个月路线图中，每延迟1个月意味着约¥10-15万的额外成本(团队工资+云服务)和¥50-100万的收入损失(假设年销5万台×单价¥400×1/12)。
五大学派项目管理分析
凯恩斯主义 ：政府应推动"适老化智能设备快速审批通道"，将SRRC/CCC认证周期从3-5月压缩至1-2月。参考医疗器械"优先审批"制度，对养老相关产品给予认证绿色通道。
奥地利学派 ：市场自发形成"快迭代"节奏——8个月路线图的本质是"最小可行时间"。不要追求完美，追求"足够好+足够快"。先上线再优化，用真实用户数据指导迭代方向。
后凯恩斯主义(Minsky) ：项目管理的最大风险是"现金流断裂"——M3-M4资金消耗最快但收入为零。建议准备6个月运营资金(¥60-80万)，确保即使产品延期也能撑到M8。Minsky时刻：当现金<¥20万时触发融资预案。
新制度经济学 ：Jira+甘特图是一套"治理制度"——降低团队协作的交易成本。但制度本身有成本(每周站会1小时×6人=6小时/周)。MVP阶段用飞书/Excel替代Jira可省¥3,000/年。
行为经济学 ："规划谬误"(Planning Fallacy)导致8个月路线图实际上通常需要10-12个月。所有硬件团队都低估了模具返工次数(平均1.5次)和认证排队时间(平均+2周)。建议在排期中加入20%缓冲。
案例深挖：Peloton的"慢" — 为什么8个月不够？
Peloton教训Peloton从概念到上市用了18个月(远超我们的8个月)，但上市后3年才实现盈利。核心原因：硬件质量不达标导致早期退货率高达15%(行业平均3%)
对我们的启示①8个月可以上市，但质量风险高。建议在DVT阶段额外增加1周"用户可用性测试"(邀请10位老人试用)，避免上市后大规模退货。②质量问题的隐性成本远高于延期成本——退货物流¥30/件×15%退货率=每台多花¥60
关键决策点如果DVT良率<90%，宁可推迟1个月上市也不要强行MP。良品率从90%提升到95%可减少5倍售后成本
建议调整原8个月路线图中，M5-M6的DVT阶段从3周延长至4周，增加1周"老人可用性测试"。总周期从8个月→9个月，但退货率从15%降至5%以下

第六步：硬件开发的独特里程碑(与纯软件不同)

  1. T0(需求冻结日)：所有功能需求在PRD中签字确认，之后任何变更需走CR(Change Request)流程。硬件开发一旦开模就无法"热更新"，冻结越早越好。
  2. T+4周(Design Review)：ID和MD设计评审，3D渲染图确认。此阶段可无成本改设计。
  3. T+8周(Tooling Release)：模具厂开工。这是"不可逆"节点——一旦开模，修改模具至少多花2-4周+几万块费用。
  4. T+14周(First Article Inspection, FAI)：第一批量产样件检验。拿着样件和图纸逐尺寸测量，公差超差立即改模。
  5. T+20周(Risk Run)：小批量试产100台，模拟真实量产流程。目标是发现产线上的"意料之外"的问题。
  6. T+26周(Mass Production Start)：首批3,000台正式量产，SRRC/CCC证书必须在此节点前到手。
