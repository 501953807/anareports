第二十章：老年人用户体验 — 大字/简单/语音/一键SOS

60岁以上用户的视力下降(老花/白内障/黄斑变性)、手指灵活性降低、学习能力弱——是硬件设计的核心约束。老年用户不是"年轻用户的慢版本"，而是完全不同的使用模式。

实操：老年用户体验设计落地 — 从原型到高保真交互

第一步：手环UI原型设计(Figma实操)

  1. 创建Figma文件：新建Frame, 尺寸280×280px(1.3寸圆形裁剪)。导入Noto Sans SC字体(已内置中文字形)。
  2. 设计主页(三功能卡片)：左滑=日期+天气, 中间=心率大数字(72bpm, 64pt粗体), 右滑=步数(3,200步) + 电池电量图标。每个卡片只放一个核心数据，不要堆砌。
  3. SOS一键页面：长电源键直接进入SOS页面，显示"紧急呼叫"四个大字(48pt)+"按SOS键接通"提示。背景色变为红色(#ef4444)。
  4. 颜色规范：背景=#0f172a(深蓝黑)，主文字=#ffffff(白色)，强调色=#06b6d4(青色)，警告色=#ef4444(红色)。

第四步：老年人可用性测试实操(SIT用户测试)

  1. 招募10位65-80岁的真实老人：通过社区养老中心/老年大学招募。准备报酬：每人¥200购物卡。确保男女各半，其中2位戴老花镜，1位视力较差。
  2. 测试任务(每个任务记录成功率和用时)：(1)"戴上手环，说出你的心跳是多少" (2)"假设你摔倒了，你怎么求救？" (3)"打开手环盒子，找到充电器并充上电" (4)"在APP上让子女看到你今天的步数" (5)"手环没电了，你怎么知道？"
  3. 评判标准：5分钟内独立完成=通过。超过5分钟或需助手提示=需要改进UI。任何一个任务通过率<80%，返回修改设计。
  4. 迭代改进：根据测试结果，修改最大的是文字太小、SOS键手感不清、充电器插口方向不明等问题。重新测试直到所有任务通过率>95%。

第六步：语音播报功能实现方案

  1. 硬件选型：手环集成微型扬声器(1W, 8Ω) + DAC(audio codec)。推荐BES2700系列(恒玄科技)，集成PDM麦克风和CLASS-D功放。支持本地TTS(文本转语音)引擎。
  2. 本地TTS引擎：使用科大讯飞离线语音合成SDK(嵌入MCU)。词库约5MB，发音自然度达到商业级。支持普通话/粤语方言识别。响应时间<300ms。
  3. 触发时机：晨间播报(天气+用药提醒+心率基线) + 事件播报(心率异常/摔倒检测到) + 交互反馈(按键音+功能确认)。
  4. 音量自适应：根据环境噪声传感器(如果有)自动调节音量。或手动三档调节(低/中/高)。默认中档(60-70dB SPL)。

第七步：老年人APP开发设计模式

老年APP的三大设计模式
①"一张纸"模式：主界面只放最重要的1-3件事(今日心率/今日步数/今日用药)，其他功能藏在二级页面。②"大图块"模式：每个功能用≥72×72dp的大按钮，间距≥16dp，避免误触。③"文字+图标"模式：所有按钮同时显示文字说明和图标(仅图标老人看不懂)。

第八步：子女端APP设计要点

子女端APP信息架构
子女端APP与老人端分离设计。①首页仪表盘：显示父母实时心率/位置/电量(一目了然的危险用红色标注) ②健康报告：可切换日/周/月视图，查看趋势 ③围栏管理：可视化创建围栏 ④告警中心：历史告警记录，可标记"已处理" ⑤设置：管理联系人、通知偏好、紧急就医信息(血型/过敏史)。
