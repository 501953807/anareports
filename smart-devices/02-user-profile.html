第二章：用户画像与应用场景 — 三个角色的三角色矛盾

智能养老产品的核心矛盾：使用者(老人)不付钱，购买者(子女)不用。设计必须同时满足三方需求——老人要"无形关怀"而非"监控感"，子女要"可靠安心"，付费方(保险/政府)要"数据价值"。

2.1 使用者 — 60-85岁老人

维度生理特征产品要求深度解读
视力老花眼，最小字体需≥14pt，高对比度手环大屏(1.14"+)、药盒LED指示灯+大字体大屏模组BOM增量约¥8-12，但可显著降低老人使用门槛，ROI极高。
听力高频听力衰减，语音需中低频、音量≥65dB药盒TTS语音提醒(60dB+)、手环震动+语音TTS语音模块BOM增量仅¥3-5，是性价比最高的适老化投入之一。
操作能力触屏精度下降，手抖大按钮+实体SOS按键，极简操作实体SOS按键可靠性远高于触屏触发，是救命功能的核心保障。
心理特征抗拒"被管理"，需要"无形关怀"外观不能像医疗设备，像普通手表心理接受度决定佩戴率——"医疗感"外观会导致老人主动摘下设备。
续航容忍度至少5天，充电本身是障碍手环5-7天、药盒7-15天续航每增加1天，BOM增加约¥2-3(电池容量)，用户满意度提升显著。
核心诉求安全(防走失)、方便(按时吃药)、不被打扰

2.2 购买者 — 40-55岁子女

维度特征产品要求深度解读
核心诉求实时知道父母在哪、有没有按时吃药核心矛盾：子女要"安心"，但过度监控会引发老人反感。需在产品设计中找到平衡点。
决策因素可靠性 > 易用性 > 价格 > 品牌可靠性排第一意味着：宁可少功能也要高稳定性。一个误报比漏报更损害信任。
价格区间手环200-500元 / 药盒100-300元愿意为"安心"支付订阅费(19-39元/月)价格敏感度低于功能可靠性。子女愿为"一次走失的避免"支付数千元，但对月度订阅较谨慎。
使用场景工作日上班，远程监控APP实时位置+用药记录+SOS告警第一时间推送异地子女是核心付费群体——同城子女可通过电话/上门解决，异地才真正依赖设备。
痛点多父母照顾压力(上有老下有小)、异地无法陪伴"三明治一代"平均需同时照顾2-4位老人，时间碎片化导致信息获取焦虑。

2.3 典型用户故事

场景故事产品介入关键价值深度解读
A：独居老人走失张奶奶78岁，轻度认知障碍，独自出门散步迷路，手机没电。GPS+北斗+基站三重定位 → 电子围栏告警 → 子女APP收实时位置 → 30分钟找到。黄金救援时间统计显示：认知障碍老人走失后2小时内找到概率>90%，超过6小时降至<30%。
B：漏服降压药李爷爷72岁，高血压+糖尿病，每天4种药。某天忘吃降压药，下午头晕入院。定时语音提醒 → 未取药 → 震动加强 → 通知子女APP → 子女电话提醒。用药依从性从40%→90%+漏服降压药使中风风险增加3-5倍。用药提醒设备的临床价值已被多项RCT研究证实。
C：跌倒+紧急呼叫王爷爷80岁，浴室滑倒，意识清醒但无法起身。手环IMU检测异常加速度 → 自动SOS → 预设联系人+位置 → 实体防水SOS按钮备用。救命跌倒是中国65岁以上人群伤害死亡的首要原因，占意外死亡的35%以上。

2.4 慢病用药场景 — 刚需驱动

疾病患者数(中国)日均用药漏服率药盒价值深度解读
高血压~2.7亿1-3种老年人依从性仅40-60%降低漏服→减少中风/心梗→节省医保高血压患者年均住院费用约¥15,000-30,000。用药提醒设备若可降低20%漏服率，单患者年省医保¥3,000-6,000。
糖尿病~1.4亿2-4种漏服→血糖波动→住院用药记录+家属提醒双重保险糖尿病并发症(肾病/失明/截肢)的长期照护成本远超设备投入，预防经济学角度ROI清晰。
心脑血管~1.1亿3-6种多药混服→相互作用风险自动分药+库存管理+过期预警多药混服导致药物相互作用事件占老年不良反应的25%。自动分药机构可从根本上降低此风险。

2.5 商业模式 — 硬件获客 + 订阅盈利

环节逻辑数据深度解读
硬件获客手环299-499元(成本~100-230元) / 药盒99-499元(成本~30-171元)硬件毛利率60-70%，但一次性收入硬件本质是获客渠道而非利润中心。参考Apple策略：硬件毛利补贴获客，服务生态赚长期利润。
订阅增值基础版免费 / 高级版19元/月 / 尊享版39元/月假设10,000用户→15%付费→ARPU 29元→月收入4.35万15%付费率假设偏保守。对比Spotify全球付费率~60%，但中国老年人付费习惯较弱，15%更务实。
LTV续费率30% × 24月 × 29元/月 = ~174元/用户174元订阅LTV > 200元硬件毛利 → 长期订阅价值超过硬件LTV计算中30%续费率是关键变量。若通过增值服务提升至50%，LTV可达¥348/用户，翻倍。
延伸收入药品电商导流 / 保险合作 / B2B养老机构批量采购手环数据→健康评分→保费优惠→双向获客保险合作是LTV跃升的关键路径——保险公司愿为降低赔付支出付费，ARPU可从¥29提升至¥200+/月。

实操：如何开展老年人智能设备用户研究 — 从招募到报告

第一步：确定调研方法论(2天)

方法适用场景样本量成本产出
深度访谈了解老人真实生活场景、痛点、用药习惯15人(60-85岁各5人)3,000元(礼品+交通)主题分析报告
可用性测试测试手环/药盒原型机的操作难度10人(配对照青年组10人)5,000元(实验室租赁+设备)SUS评分+改进建议
问卷调查量化用户需求优先级、付费意愿200份有效样本2,000元(问卷星专业版)数据交叉分析报告
家庭入户观察记录老人日常行为模式(吃药/起居/外出)5个家庭×7天8,000元(调研员薪资)行为日志+时间线

第二步：老人深度访谈提纲(10题模板)

  1. "您平时一天吃几次药？都是什么时间？" — 摸底用药频率，判断需要分几格药盒
  2. "上次忘记吃药是什么时候？后来怎么发现的？" — 找漏服场景的真实触发机制
  3. "您最近一次出门迷路/找不到路是什么时候？" — 评估走失风险的真实发生频率
  4. "如果有个手表能提醒您吃药、告诉孩子您在哪，您觉得有用吗？" — 测试接受度(注意措辞不能是"监控")
  5. "您的子女多久给您打一次电话？最担心他们什么事？" — 定位情感需求层次
  6. "您戴过什么电子产品？觉得哪里不方便？" — 了解既有电子产品的使用门槛
  7. "充电对您来说容易吗？一般几天充一次电？" — 评估续航目标设定(至少7天)
  8. "如果这个设备能帮您测心率、测血压，您会相信上面的数字吗？" — 测试健康数据信任度
  9. "您希望这个设备长什么样？像手表还是像医疗器材？" — ID设计关键输入(要像时尚手表)
  10. "如果每个月需要付30元用这个服务，您觉得值吗？谁付这笔钱？" — 付费意愿验证(老人vs子女)

第三步：用户招募渠道指南

渠道操作方式单个成本到达人数质量
社区居委会联系街道办，在社区公告栏贴海报，现场招募50元/人(礼品)每个社区10-20人高(真实的社区老人)
公园晨练群体早7点在社区公园蹲点，发小礼品邀请填写问卷20元/人(鸡蛋+纸巾)每次50-100人中高(活跃老人，可能低估需求)
老年大学联系校长，在课堂上做5分钟宣讲+扫码报名500元/场(场地+咖啡)每期30-50人高(认知水平较高的老人)
医院内分泌科/心内科等待候诊区发放调研问卷(需医院伦理委员会批准)1000元/天(研究员+礼品)每次20-30人高(慢病患者的用药需求最强)
微信群/朋友圈在"子女照顾群""照护者联盟"群中发红包问卷5元/人(红包)取决于群活跃度中(受访者是子女本人)

第四步：建立用户画像卡片(模板)

画像A：独居奶奶型

姓名：张秀兰，78岁，独居，轻度高血压。子女在北京工作。每天自行服药2次。去年走失2次(每次2小时找回)。需求排序：防走失(SOS) > 用药提醒 > 心率监测。

设计要点：大字体(18pt+)、实体SOS键、电子围栏、超长续航

画像B：照护妈妈型

姓名：李薇，45岁，企业中层，母亲72岁住同城。母亲有3种慢性药(降压+降糖+降脂)。经常担心母亲漏服。工作日忙没时间打电话。需求：远程监控 + 用药记录通知 + 紧急联络。

设计要点：简洁APP界面、微信消息推送、用药统计周报

第五步：可用性测试设置(家庭实验室)

  1. 准备设备：用3D打印外壳+Arduino模拟手环原型，LCD屏显示心率数字，USB供电。药盒用乐高搭建分药机构原型。总成本约500元。
  2. 测试任务：给每个老人3个任务：(1)按下SOS键(2)读出屏幕上的心率(3)打开药盒拿一片药。记录完成时间和错误次数。
  3. Think-aloud录音：请老人边操作边说出想法。"你看到这个按钮想按吗？为什么？你觉得它会做什么？"
  4. 量表评分：完成后让老人用1-5分打分：(1)好不好用 (2)会不会嫌麻烦 (3)愿不愿意戴出去 (4)愿不愿意告诉朋友
  5. 数据分析：SUS(System Usability Scale) = Σ(每项得分×2.5)。目标：SUS > 60(及格线)，>75(优秀)

真实洞察案例：充电恐惧
在某社区访谈中发现，43%的老人说"手表电量少了不知道怎么办"——他们不敢戴手环因为怕电池耗尽后无法联系子女。这个洞察直接改变了产品设计：把手环续航从目标5天提升到7天，并在电量低于20%时用TTS语音提醒("张奶奶，手环该充电了")，而非仅仅在手机APP上显示一个数字图标。

第七步：用户旅程地图绘制方法

阶段老人行为情绪曲线触点机会点
收到礼物子女送上手环，帮佩戴+2(惊喜)包装盒上手环外观包装内附大字版"使用指南"

SOS触发意外跌倒后一键求助未知实体SOS按钮必须保证一键直达，不能有任何前置步骤

第八步：场景验证清单——产品上线前必做的事

  1. 模拟走失场景：戴上手环，去一个陌生地方走2公里，看APP定位是否能准确跟随。
  2. 模拟漏服场景：设置药盒提醒时间，故意不吃药，看子女是否会收到通知和二次提醒。
  3. 模拟低电量场景：把电池放电到10%，看手环是否会发出低电量语音提醒，APP是否有足够提前量。
  4. 模拟老人视角：让团队里最年轻的成员(22岁)戴上老花镜来操作手环和APP，记录每一步卡点。
  5. 模拟子女焦虑：让测试者连续3天不在家，期间手环发出多次SOS告警，感受"父母在 danger"的紧迫感。
  6. 极端网络测试：在电梯/地下室等无信号场景，定位数据和SOS告警是否会缓存、有信号后是否自动补传。
用户研究ROI公式
投入3万元做用户研究(访谈+可用性测试+入户观察)，可能避免30万元的后期设计返工。一个典型的教训：我们原以为需要"大屏幕"，但测试发现老人只需要"大按钮+语音播报"。省下了一块昂贵的AMOLED屏幕预算。
细微信号 — "三明治一代"的付费意愿拐点
2024年《中国家庭照护负担报告》显示：40-55岁子女平均每天花费1.8小时照护老人，经济支出占家庭收入的23%。更关键的是，"异地照护"群体(父母与子女不在同一城市)的付费意愿显著高于同城群体——月均可支配预算从¥50跃升至¥200+。这一结构性变化意味着：我们的产品定位应优先瞄准"异地子女"这一高付费意愿群体。
多学派交叉解读 — 用户画像背后的经济学逻辑
凯恩斯主义
有效需求不足是核心问题：老人有安全需求但无支付能力，子女有支付能力但非使用者。政府补贴(每台100-300元)可填补这一需求缺口，通过财政支出创造有效需求，产生乘数效应。
信息经济学
子女与老人之间存在严重的信息不对称——老人隐瞒不适、子女无法实时掌握健康状态。手环+药盒+APP构成的数据链路本质上是降低信息不对称的机制，减少逆向选择和道德风险。
公共选择理论
政府采购智能养老设备的决策链涉及多层利益相关者(民政/财政/养老院)。决策者追求政治效用最大化而非社会福利最大化，可能导致"重采购轻运营"现象。建议产品设计时考虑B2G场景的特殊性。
家庭经济学(Becker)
加里·贝克尔的家庭生产函数理论：子女用金钱(买设备)替代时间(亲自照顾)来生产"父母安全"这一家庭公共品。当子女时间机会成本上升(升职/加班)，设备替代效应增强，市场自然扩张。
行为经济学
损失厌恶驱动购买决策：子女为父母买设备不是出于"获得安心"的收益预期，而是"避免出事"的损失规避。营销话术应侧重"万一走失怎么办"而非"有了设备多好"。框架效应可使转化率提升2-3倍。
案例深挖 — 小寻T5s Pro的用户留存教训
背景：小寻T5s Pro是小米生态链企业出品，299元老人GPS手表，功能全面(多重定位+SOS+电子围栏+视频通话)，但用户留存率仅约35%(6个月后仍在活跃)。核心问题：①需SIM卡月费(8-15元/月)增加持续使用门槛 ②APP界面不够适老化，老人子女双端体验割裂 ③缺乏"不可替代"的内容服务——只有定位没有健康/用药维度。启示：我们的差异化策略应是：免卡方案(eSIM内置)+微信生态免下载+手环+药盒双设备联动，形成小寻无法复制的生态壁垒。
发散引申链 — 从用户画像到产品演进路径
老人-使用者 → 子女-购买者 → 保险/政府-付费方 → 医院/社区-服务方 | C端订阅(Year 1) → B2B养老机构(Year 2) → B2G政府采购(Year 3) → 保险精算合作(Year 4+)
战略建议 — 三角色矛盾的产品解法
解决"使用者不付钱、购买者不用"的核心矛盾，产品设计必须遵循三个原则：第一，老人端做减法——所有功能默认开启，无需老人操作；第二，子女端做加法——APP提供丰富数据和可控选项，满足"掌控感"心理需求；第三，付费方做价值化——向保险/政府展示可量化的ROI(如"每投入1元设备费，节省医保支出5元")。三端体验的平衡是产品成败的关键。
