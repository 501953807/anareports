第二章：用户画像与应用场景 — 三个角色的三角色矛盾

智能养老产品的核心矛盾：使用者(老人)不付钱，购买者(子女)不用。设计必须同时满足三方需求——老人要"无形关怀"而非"监控感"，子女要"可靠安心"，付费方(保险/政府)要"数据价值"。

2.1 使用者 — 60-85岁老人

维度生理特征产品要求
视力老花眼，最小字体需≥14pt，高对比度手环大屏(1.14"+)、药盒LED指示灯+大字体
听力高频听力衰减，语音需中低频、音量≥65dB药盒TTS语音提醒(60dB+)、手环震动+语音
操作能力触屏精度下降，手抖大按钮+实体SOS按键，极简操作
心理特征抗拒"被管理"，需要"无形关怀"外观不能像医疗设备，像普通手表
续航容忍度至少5天，充电本身是障碍手环5-7天、药盒7-15天
核心诉求安全(防走失)、方便(按时吃药)、不被打扰

2.2 购买者 — 40-55岁子女

维度特征产品要求
核心诉求实时知道父母在哪、有没有按时吃药
决策因素可靠性 > 易用性 > 价格 > 品牌
价格区间手环200-500元 / 药盒100-300元愿意为"安心"支付订阅费(19-39元/月)
使用场景工作日上班，远程监控APP实时位置+用药记录+SOS告警第一时间推送
痛点多父母照顾压力(上有老下有小)、异地无法陪伴

2.3 典型用户故事

场景故事产品介入关键价值
A：独居老人走失张奶奶78岁，轻度认知障碍，独自出门散步迷路，手机没电。GPS+北斗+基站三重定位 → 电子围栏告警 → 子女APP收实时位置 → 30分钟找到。黄金救援时间
B：漏服降压药李爷爷72岁，高血压+糖尿病，每天4种药。某天忘吃降压药，下午头晕入院。定时语音提醒 → 未取药 → 震动加强 → 通知子女APP → 子女电话提醒。用药依从性从40%→90%+
C：跌倒+紧急呼叫王爷爷80岁，浴室滑倒，意识清醒但无法起身。手环IMU检测异常加速度 → 自动SOS → 预设联系人+位置 → 实体防水SOS按钮备用。救命

2.4 慢病用药场景 — 刚需驱动

疾病患者数(中国)日均用药漏服率药盒价值
高血压~2.7亿1-3种老年人依从性仅40-60%降低漏服→减少中风/心梗→节省医保
糖尿病~1.4亿2-4种漏服→血糖波动→住院用药记录+家属提醒双重保险
心脑血管~1.1亿3-6种多药混服→相互作用风险自动分药+库存管理+过期预警

2.5 商业模式 — 硬件获客 + 订阅盈利

环节逻辑数据
硬件获客手环299-499元(成本~100-230元) / 药盒99-499元(成本~30-171元)硬件毛利率60-70%，但一次性收入
订阅增值基础版免费 / 高级版19元/月 / 尊享版39元/月假设10,000用户→15%付费→ARPU 29元→月收入4.35万
LTV续费率30% × 24月 × 29元/月 = ~174元/用户174元订阅LTV > 200元硬件毛利 → 长期订阅价值超过硬件
延伸收入药品电商导流 / 保险合作 / B2B养老机构批量采购手环数据→健康评分→保费优惠→双向获客
