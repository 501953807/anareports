23.5 手把手：TensorFlow Lite Micro 模型训练与边缘部署

步骤一：数据采集与标注

跌倒检测数据集构建方案
推荐方案：公开数据集 + 自建采集。开源数据集可用 WISDM (1,092用户, 6类活动) 和 UCI-HAR (108用户, 12,000段窗口)。自建采集需要找30名志愿者在不同场景(沙发、卧室、厨房)模拟跌倒20次 + 日常活动50次。关键：跌倒场景要覆盖"向前摔倒""向后摔倒""侧滑倒地"三类。
数据集目录结构与标注格式 
dataset/
  └── fall_detection/
      ├── raw/                    # 原始 IMU 数据 (CSV)
      │   ├── volunteer_001_fall_01.csv
      │   ├── volunteer_001_daily_walk.csv
      │   └── ...
      ├── annotated/              # 标注后 (HDF5 格式, TF 原生)
      │   ├── train.tfrecord      # 训练集 70%
      │   ├── val.tfrecord        # 验证集 15%
      │   └── test.tfrecord       # 测试集 15%
      └── metadata.json           # 志愿者年龄/体重/摔倒次数统计

步骤二：训练 1D-CNN 跌倒检测模型

Python 训练脚本 
# train_fall_detector.py
import tensorflow as tf
import numpy as np

# 输入形状: (batch, window_length=50, channels=6)
# 6通道: accelerometer(x,y,z) + gyroscope(x,y,z)
INPUT_SHAPE = (50, 6)
NUM_CLASSES = 2  # fall / not_fall

model = tf.keras.Sequential([
    tf.keras.layers.Input(shape=INPUT_SHAPE),
    tf.keras.layers.Conv1D(64, 3, padding='same', activation='relu'),
    tf.keras.layers.BatchNormalization(),
    tf.keras.layers.MaxPooling1D(2),

    tf.keras.layers.Conv1D(128, 3, padding='same', activation='relu'),
    tf.keras.layers.BatchNormalization(),
    tf.keras.layers.MaxPooling1D(2),

    tf.keras.layers.GlobalAveragePooling1D(),
    tf.keras.layers.Dense(32, activation='relu'),
    tf.keras.layers.Dropout(0.3),
    tf.keras.layers.Dense(NUM_CLASSES, activation='softmax')
])

model.compile(
    optimizer=tf.keras.optimizers.Adam(learning_rate=1e-3),
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)

model.summary()
# Total params: ~130,000 → 量化后可压缩到 ~65KB

# 训练(早停防止过拟合)
callback = tf.keras.callbacks.EarlyStopping(
    monitor='val_loss', patience=5, restore_best_weights=True)

history = model.fit(
    train_data, train_labels,
    validation_data=(val_data, val_labels),
    epochs=50, batch_size=32, callbacks=[callback])

# 保存 TFLite 格式(边缘端专用)
converter = tf.lite.TFLiteConverter.from_keras_model(model)
converter.optimizations = [tf.lite.Optimize.DEFAULT]
tflite_model = converter.convert()
with open('fall_detector.tflite', 'wb') as f:
    f.write(tflite_model)
print(f'Model size: {len(tflite_model) / 1024:.1f} KB')
# 预期输出: Model size: 198.4 KB (满足 MCU 256KB Flash 限制)

步骤三：模型量化与压缩

量化方案模型大小推理速度精度损失适用场景
FP32(原始)~200KB慢(基准)0%云端训练/调试
INT8 全量化~50KB4x 加速-1.5%Cortex-M4F MCU
FLOAT16~100KB1.5x 加速-0.3%Cortex-A 系列手机
剪枝(50%权重为零)~80KB2x 加速-2.0%内存紧张场景
量化实战经验
实测数据：INT8 量化后准确率从 96.5% 降到 95.0%，但推理速度从 12ms 降到 3ms，功耗降低 60%。对于跌倒检测这种"宁可误报不可漏报"的应用，牺牲1.5%精度换取 4x 速度提升是完全值得的。建议设置两个阈值：MCU 用 0.6 快速触发 → APP 用 0.8 做二次确认。

步骤四：Cortex-M4 端部署

STM32 嵌入式推理代码 
// fall_detector.c (STM32 + TFLite Micro)
#include "tensorflow/lite/micro/micro_mutable_op_resolver.h"
#include "tensorflow/lite/micro/micro_interpreter.h"
#include "model.h"  // 编译生成的 fall_detector.cc

constexpr int kTensorArenaSize = 32 * 1024;  // 32KB RAM
uint8_t tensor_area[kTensorArenaSize];

static tflite::MicroInterpreter interpreter(
    GetModel(), resolver, tensor_area, kTensorArenaSize);

void detect_fall(int16_t* imu_buffer, int length) {
  TfLiteTensor* input = interpreter.input(0);
  float* input_data = input->data.f;

  // 1. 数据归一化(-1~1)
  for (int i = 0; i < length * 6; i++) {
    input_data[i] = imu_buffer[i] / 32768.0f;
  }

  // 2. 运行推理
  TfLiteStatus status = interpreter.Invoke();

  // 3. 读取输出概率
  float* output = interpreter.output(0)->data.f;
  float fall_prob = output[0];  // fall 类概率
  float normal_prob = output[1]; // not_fall

  if (fall_prob > 0.60f) {
    trigger_fall_alert();  // 触发跌倒告警
  }
}
