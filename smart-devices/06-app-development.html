实操：APP开发完整教程 — 从环境搭建到上线发布

第一步：Flutter项目初始化(30分钟搞定)

  1. 下载Flutter SDK: 访问 flutter.dev/downloads，下载macOS版flutter_macos_v3.24.zip，解压到~/development/flutter。
  2. 配置环境变量: 在~/.zshrc中添加export PATH="$PATH:~/development/flutter/bin"，然后source ~/.zshrc。
  3. 运行flutter doctor：检查Android SDK、Xcode、Dart插件(VS Code)是否就绪。
  4. 创建项目: flutter create wellness_watch && cd wellness_watch。
  5. 加入核心依赖(pubspec.yaml): ble_platform_detection: ^1.0、flutter_blue_plus: ^1.33、http: ^1.2、shared_preferences: ^2.2、geolocator: ^12.0、amap_map_fluttify: ^0.0.14。

第三步：高德地图集成(轨迹显示+电子围栏)

功能技术方案耗时要点
实时定位高德地图Flutter SDK(AMapFlutter) + Geolocator插件3天需要申请高德KEY，Android在AndroidManifest配置，iOS在Info.plist配置
轨迹回放AMapPolyline绘制折线，按timestamp排序插入坐标点2天每5秒一个点，过度用线性插值保持流畅
电子围栏AMapPolygon + 多边形点包含算法(Point-in-Polygon)2天射线法：从测试点发水平射线，统计与多边形边的交点数
逆地理编码高德GCJ-03逆编码API：坐标→地址文本1天"39.9042,116.4074" → "北京市朝阳区某某小区"

第四步：推送通知多渠道配置

  1. APNs(iOS)：在Apple Developer Portal创建Push Notification Certificate(.p12)，上传到阿里云推送控制台。
  2. 厂商通道(Android)：注册小米推送(XPS)、华为推送(HPS)、OPPO Push、vivo Push四个厂商通道。一个APP只需配一次，可覆盖95% Android设备。
  3. 微信模板消息：在微信公众平台(mp.weixin.qq.com)注册服务号，配置"用药提醒"和"SOS告警"两个模板。用户授权后通过微信推送。
  4. 短信兜底：接入阿里云短信服务(aliyun.com/sms)，P0级告警(跌倒/越界)同时触发APP推送+短信。

发布前的必做测试
1) 在5种不同机型上跑完所有功能(最低配安卓5.1 + 最新iOS)。2) BLE断连重连测试：故意关闭蓝牙看APP是否正确弹出"正在连接"。3) 弱网测试：用Charles设置3G慢速网络，看定位数据是否缓存离线后补传。4) 安装卸载测试：卸载重装后设备需能重新配对。

第七步：APP灰度发布策略(先1%后100%)

  1. 内部测试版：通过Firebase Test Lab或Google Play Internal Testing，先对内部分发，2人开发团队连续使用3天。
  2. Alpha灰度(1%)：在App Store设为"TestFlight"测试，限定邀请码。收集Crashlytics崩溃率，目标<0.5%。
  3. Beta灰度(10%)：华为/小米应用市场设"先锋体验官"计划，用户报名后优先安装。收集用户反馈问卷(用Typeform，¥299/月)。
  4. 全量发布：各项指标达标后全量上架。发布后24小时内密切监控崩溃率和闪退率。
