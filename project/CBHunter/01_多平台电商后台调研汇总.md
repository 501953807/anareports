```
│   └── status: enum
├── platform_extension（平台扩展字段，JSONB）
│   ├── platform_code: shopee/tiktok temu
│   ├── ssp_id / sps_score / spu_id / skc_id
│   ├── supply_price_cny / retail_price_compliance
│   ├── warehouse_group / stocking_logic
│   └── custom_fields: jsonb
└── country_config（国别配置）
    ├── currency_code
    ├── tax_rate
    └── logistics_channels: jsonb
```

#### 6.8.3 SKU/变体管理设计

支持两种模式：
1. **简单模式（妙手/Shopee/TikTok风格）**：规格一、规格二两行定义
2. **三层模式（TEMU风格）**：SPU → SKC → SKU

SKU生成规则参考妙手ERP：默认 `货源ID + 规格1 + 规格2`，分隔符可配，超过100个SKU时提示而非自动删除。

#### 6.8.4 图片编辑器集成方案

分三阶段实现：
- **第一阶段（MVP）**：内置轻量级编辑器，支持上传/预览/裁剪/旋转/水印
- **第二阶段**：集成易可图或类似SaaS服务，支持批量编辑（10张/次）、AI设计、智能抠图
- **第三阶段**：根据用户需求自研完整编辑器，支持更多电商专用功能

#### 6.8.5 批量编辑功能设计

参考妙手ERP的批量编辑弹窗：

| 功能 | 实现方式 |
|------|----------|
| 批量标题优化 | AI自动优化/关键词随机组合 |
| 批量图片编辑 | 调用图片编辑器批量处理 |
| 批量SKU修改 | 表格内直接编辑 |
| 批量价格调整 | 公式计算（如统一加价10%） |
| 批量类目设置 | 类目选择器 |

### 7.2 CBHunter 设计启示（更新）

1. **统一底座 + 平台适配器** 是最优架构（妙手已验证）
2. **采集箱/待发布池** 是高效的多平台商品管理模式
3. **动态表单** 必须支持类目选择后加载差异化字段
4. **批量操作** 应前置到采集/编辑阶段
5. **只读优先** 是合规底线，涉及资金和发布的操作需人工确认
6. **国别差异** 必须在数据层隔离，不能硬编码在前端
7. **弹窗式编辑** 优于独立页面编辑（参考妙手ERP），保持上下文、支持批量操作
8. **图片编辑器解耦** 通过iframe嵌入，支持批量编辑和后期替换
9. **两层+三层混合SKU** 兼容普通平台和TEMU全托管模式
