# 功能开发方案总文档

> **项目**: CBHunter — 多平台跨境电商卖家后台  
> **版本**: v1.0  
> **日期**: 2026-07-20  
> **核心用户**: 跨境卖家/运营团队  
> **优先级平台**: Shopee（东南亚站点优先）

---

## 一、总体架构方案

### 1.1 核心架构决策：混合框架法（推荐）

#### 方案对比

| 维度 | 方案A：统一底座 + 平台适配器（推荐） | 方案B：每个平台独立开发一套发布器 | 方案C：纯统一底座无扩展 |
|------|-------------------------------------|----------------------------------|------------------------|
| 开发效率 | 高：公共逻辑复用，差异化最小化 | 低：每平台重复开发商品/订单/资金模块 | 中：一次开发但扩展困难 |
| 维护成本 | 中：公共层统一维护，适配器独立维护 | 高：每平台独立维护，bug修复需N份 | 低但不可持续 |
| 扩展性 | 高：新增平台只需新增适配器 | 低：每平台独立代码库 | 极低：平台差异无法表达 |
| 适配Shopee | ✅ 完整支持SSP/Hot Listing/4步上架 | ✅ 天然支持 | ❌ 无法表达SSP等独有概念 |
| 适配TikTok | ✅ SPS/CRM/直播生态可独立建模 | ✅ 天然支持 | ❌ 无法表达SPS门槛 |
| 适配Temu | ✅ SPU/SKC/SKU三层+供货价模型可独立建模 | ✅ 天然支持 | ❌ 无法表达全托管模式 |
| 适配妙手 | ✅ 采集箱/批量操作/三态管理可映射 | ❌ 妙手是工具不是平台 | ❌ 无法表达采集箱 |

#### 最终架构：混合框架法

```
┌─────────────────────────────────────────────────────────┐
│                    CBHunter 前端应用                      │
├─────────────────────────────────────────────────────────┤
│                   共享 UI 组件层                          │
│  商品列表 / 订单卡片 / 资金面板 / 表单引擎 / 筛选器        │
├─────────────────────────────────────────────────────────┤
│                 业务逻辑层（统一底座）                      │
│  商品管理 · 订单管理 · 资金管理 · 营销管理 · 物流管理        │
│  店铺管理 · 权限管理 · 通知中心 · 数据看板                  │
├─────────────────────────────────────────────────────────┤
│              平台适配器层（Platform Adapter Layer）         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐    │
│  │ Shopee   │ │ TikTok   │ │ Temu     │ │ Miaoshou │    │
│  │ Adapter  │ │ Adapter  │ │ Adapter  │ │ Adapter  │    │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘    │
│  字段映射 · 流程编排 · 状态机 · 权限策略 · 国别配置         │
├─────────────────────────────────────────────────────────┤
│              数据层（Unified Data Model）                  │
│  标准字段 · 扩展字段 · 国别差异字段 · 平台元数据             │
└─────────────────────────────────────────────────────────┘
```

**核心原则**:
- **统一底座**负责所有平台通用的业务流程和UI交互
- **平台适配器**负责将平台特有概念映射到统一模型，以及处理平台特有的API/字段/流程
- **数据层**通过标准字段 + 扩展字段实现跨平台数据互通

### 1.2 多平台发布功能设计

#### 1.2.1 发布流程抽象

所有平台的商品发布流程抽象为三个阶段：

```
采集/新建 → 编辑/映射 → 预览/发布
```

**阶段1 — 采集/新建**:
- 来源：手动创建、链接采集、1688采集、关键词搜索、整店采集
- 产出：统一商品对象（Unified Product Object, UPO）

**阶段2 — 编辑/映射**:
- 字段映射：UPO → 目标平台字段
- 动态表单：根据类目/平台加载差异化字段
- 预览：目标平台商品卡片预览

**阶段3 — 发布/同步**:
- 提交到目标平台
- 状态跟踪：未发布 → 定时发布 → 已发布 / 失败
- 同步：平台字段变更回写CBHunter

#### 1.2.2 多平台发布调度策略

| 策略 | 适用场景 | 说明 |
|------|----------|------|
| 一键多平台发布 | 商品标准化程度高时 | 同一商品同时发布到多个平台 |
| 逐平台差异化发布 | 各平台字段差异大时 | 同一商品针对不同平台做差异化编辑后分别发布 |
| 批量发布 | 采集箱中待发布商品 | 选中多个商品一次性发布到指定平台 |
| 定时发布 | 运营排期需求 | 设置发布时间后自动发布 |

#### 1.2.3 编辑入口设计：弹窗式编辑（推荐）

基于四平台编辑页面深度调研，CBHunter采用**弹窗式编辑**而非独立页面编辑，理由如下：

| 维度 | 独立页面编辑（Shopee/TikTok/TEMU） | 弹窗式编辑（妙手ERP） |
|------|-----------------------------------|---------------------|
| 批量操作 | 不支持 | ✅ 支持 |
| 上下文保持 | 需离开列表页 | ✅ 不离开列表页 |
| 编辑效率 | 低（每次跳转） | 高（直接编辑） |
| 图片编辑器集成 | 困难 | ✅ iframe嵌入 |
| 平台适配灵活性 | 固定流程 | ✅ 动态字段渲染 |

**编辑弹窗架构**：

```
产品列表页
└── 编辑弹窗（ElDialog / Ant Design Modal）
    ├── 左侧：步骤导航（Shopee风格）或 标签页（妙手风格）
    ├── 右侧：表单内容区
    │   ├── 基本信息标签页
    │   ├── 销售资料标签页（含SKU/变体管理）
    │   ├── 图片管理标签页（含图片编辑器）
    │   ├── 物流设置标签页
    │   └── 其他属性标签页
    └── 底部操作栏：保存草稿 | 创建/更新 | 发布
        └── iframe: 图片编辑器组件（可配置接入易可图或自研）
```

**弹窗层级结构**：

```
主弹窗（产品编辑）
├── 步骤导航栏（固定左侧）
├── 内容区（动态渲染）
│   ├── 基本信息标签页
│   │   ├── 商品图片上传区（最多9张，拖拽排序）
│   │   ├── 商品视频上传区
│   │   ├── 商品名称输入框（字符计数）
│   │   ├── 类目选择器（三级联动）
│   │   └── 商品描述富文本编辑器
│   │
│   ├── 销售资料标签页
│   │   ├── SKU管理表格（动态行列）
│   │   │   ├── 规格一输入
│   │   │   ├── 规格二输入
│   │   │   ├── SKU图片上传
│   │   │   ├── 价格/库存/货号
│   │   │   └── 体积/重量（TEMU）
│   │   ├── 定价区域
│   │   │   ├── Shopee/TikTok: 售价 + 原价
│   │   │   └── TEMU: 申报价格(CNY) + 开款价格状态
│   │   └── 批量导入/导出
│   │
│   ├── 图片管理标签页

│   │   ├── SKU图片管理（按SKU分组显示）

│   │   └── 图片编辑器iframe

│   ├── 物流设置标签页

│   │   ├── 重量/尺寸
│   │   └── 发货天数/备货仓组（TEMU）
│   │
│   └── 其他属性标签页
│       ├── 品牌选择

│       ├── 合规信息（TEMU强制：资质/实拍图/合规信息）
```

- 树状视图：按类目/分组展开，适合类目丰富的平台（Shopee/Temu）

- 卡片视图：图片+关键信息，适合快速浏览（Shopee风格）

**列表视图核心列**:

| 列 | 通用字段 | Shopee特有 | TikTok特有 | Temu特有 |

| 操作 | 编辑、推广、复制 | 链接标准商品、Hot Listing | View Details、Process | 编辑、维护货号、申请修改信息 |

### 1.4 多国家市场差异化解决方案

#### 1.4.1 数据层设计

```
Product (标准表)
├── standard_fields (JSONB): 标题、描述、图片、视频、类目路径
├── sku_table (SKU子表)
│   ├── sku_id
│   ├── variant_attributes (JSONB): 颜色、尺寸等
│   ├── price_cny
│   ├── stock_qty
│   └── dimensions (长×宽×高)
├── platform_extension (平台扩展表)
│   ├── platform_code (shopee/tiktok temu)
│   ├── country_code (PH/SG/MY/...)
│   ├── ssp_id (Shopee标准商品编号)
│   ├── hot_listing_status (Shopee Hot Listing状态)
│   ├── sps_score (TikTok卖家体验分)
│   ├── supply_price_cny (Temu供货价)
│   ├── retail_price_compliance (Temu零售价合规)
│   └── custom_fields (JSONB): 各平台自定义字段
└── country_config (国别配置表)
    ├── country_code
    ├── currency_code (PHP/SGD/MYR/...)
    ├── currency_symbol (₱/S$/RM/...)
    ├── tax_rate
    ├── payment_methods (JSONB)
    ├── logistics_channels (JSONB)
    ├── category_tree_version
    └── compliance_requirements (JSONB)
```

#### 1.4.2 业务层设计

**定价引擎**:
- 统一以CNY存储基础价格
- 国别配置表定义汇率和货币显示格式
- Temu供货价→零售价转换在平台适配器层完成
- Shopee/TikTok直接存储卖家定价，国别只影响显示

**订单费用引擎**:
- 统一费用项：商品金额、运费、平台佣金、支付手续费、退款扣除
- 国别差异：各国支付方式不同导致的手续费率不同
- 结算货币：Shopee/TikTok按站点本地币结算，Temu按CNY结算

**类目体系**:
- 类目树按平台+国别存储，支持动态加载
- 类目属性按平台+国别+类目级别存储
- 选择类目后动态渲染对应字段

#### 1.4.3 前端渲染设计

**国别感知组件**:
- `CurrencyDisplay`: 根据当前站点自动格式化货币
- `CategorySelector`: 按平台+国别加载类目树
- `FieldRenderer`: 根据类目属性动态渲染表单字段
- `OrderStatusBadge`: 根据平台状态机渲染状态标签

## 二、标准化模块功能需求

### 2.1 商品管理模块

#### 2.1.1 商品列表页

**页面布局**:
- 顶部：平台选择 + 店铺/站点选择 + 发品额度显示
- 左侧：类目筛选 + 状态标签 + 快捷筛选
- 主体：商品列表（表格/卡片/树状切换）
- 右侧：商品详情抽屉（点击商品后从右侧滑出）

**筛选条件**:
- 商品名称（模糊搜索，支持Excel粘贴）
- 商品分类（下拉选择）
- 商品属性（下拉选择）
- 货号/SKU ID/SPU ID（精确查询）
- 状态标签（全部/在售中/未发布/已下架/已删除）
- 价格区间
- 库存区间
- 创建时间范围

**批量操作**:
- 批量发布、批量编辑、批量设置类目、批量设置售价
- 引用模板、分组、创建产品
- 下载查询结果、批量删除、更多

**排序**:
- 按推荐排序、按最近更新时间、按销量、按价格、按库存

#### 2.1.2 商品创建/编辑页

**通用流程**:
1. 选择类目（必填）
2. 填写基本信息（标题、图片、视频、描述）
3. 填写销售资料（SKU、定价、库存、变体）
4. 填写运费/物流（重量、尺寸、物流渠道）
5. 填写其他（自定义属性、合规信息、认证要求）
6. 预览 + 发布

**字段分组**:

| 分组 | 字段 | 必填 | 说明 |
|------|------|------|------|
| 商品图片 | 主图、附图(最多9张) | 是 | 支持预览、拖拽排序 |
| 商品视频 | 视频上传 | 否 | Max 30MB, 10-60s, MP4 |
| 商品名称 | 标题 | 是 | 最多180字符 |
| 商品描述 | 富文本/纯文本 | 是 | 最多5000字符 |
| 类目 | 三级类目选择 | 是 | 选择后动态加载字段 |
| 品牌 | 品牌选择 | 否 | 部分平台必填 |
| SKU管理 | 规格、价格、库存 | 是 | 支持批量导入 |
| 定价 | 售价、原价、供货价 | 是 | Temu填供货价 |
| 物流 | 重量、尺寸、渠道 | 是 | 按国别加载物流选项 |
| 合规 | 资质、实拍图、认证 | 按需 | Temu强制，Shopee部分类目 |

**底部操作按钮**:
- 取消
- 保存草稿
- 创建/更新
- 创建并发布/更新并发布

#### 2.1.3 商品状态机

```
草稿 → 待审核 → 已发布 → 已下架
  ↓       ↓           ↓
已删除   审核驳回     违规删除
              ↓
           重新提交
```

**各平台状态映射**:

| 状态 | Shopee | TikTok Shop | Temu |
|------|--------|-------------|------|
| 草稿 | — | — | 商品草稿 |
| 待审核 | 审查中 | — | 提交成功 |
| 已发布 | 架上商品 | To Ship/Completed | 在售中 |
| 已下架 | 未上架 | Cancelled | 已下架/已终止 |
| 违规 | 违规/删除 | — | 已废弃商品 |
| 未发布到站点 | — | — | 未发布到站点 |

### 2.2 营销中心模块

#### 2.2.1 Shopee 营销活动

**折扣活动**:
- 类型：套装优惠、我的折扣活动、分类折扣
- 字段：活动名称、商品范围、折扣规则、活动时间、商品数量
- 操作：编辑、复制、分享、详情、订单、数据

**店内秒杀 (Flash Sale)**:
- 流程：选时段 → 添加商品 → 显示在商店页面
- 统计：销售、订单数、买家数、CTR
- 准入：需达到特定标准

**优惠券**:
- 8种类型：店铺券、商品券、非公开券、直播券、短视频券、新买家券、回购买家券、关注礼券
- 字段：活动名称、优惠券码、类型、适用范围、目标买家、折扣金额、使用总数、已使用、期间
- 操作：编辑、复制、分享、订单、结束、详情

**联盟营销**:
- 商品级佣金设置
- 独立佣金面板

#### 2.2.2 TikTok Shop 营销活动

**聊天营销 (CRM)**:
- 访问条件：SPS ≥ 3.5 且 GMV > 0
- 一次性计划模板：分享热销商品、新品公告、直播促销、粉丝转化、复购推动、不活跃客户赢回、评价邀请
- 自动计划模板：直播推广、开播提醒、降价提醒、购物车挽回、结账挽回、售后感谢
- 配额管理：本周总配额、剩余配额
- 数据统计：聊天GMV、聊天订单数、聊天转化客户数

**广告营销**: TikTok Ads 对接
**联盟营销**: `/affiliate/landing`

#### 2.2.3 Temu 营销活动

- 平台统一下发，商家侧操作有限
- 爆款跟卖、机会商品推荐
- 流量扶持领取
- 传视频得免费流量加权

### 2.3 订单管理模块

#### 2.3.1 订单列表页

### 3.2 差异化配置存储方案

**配置存储策略**:
- 平台级配置存储在 `platform_configs` 表（类目树、物流渠道、支付方式、合规要求）
- 国别级配置存储在 `country_configs` 表（货币、汇率、税费、本地化文案）
- 商品级扩展字段存储在 `product_extensions` JSONB字段
- SKU级扩展字段存储在 `sku_extensions` JSONB字段

**配置更新机制**:
- 平台类目树变更时，通过适配器拉取最新类目树并缓存
- 国别配置变更时，按站点独立更新
- 平台规则变更时，通过Webhook或定期轮询检测

### 3.3 动态表单渲染逻辑

**渲染引擎设计**:

```
类目选择 → 加载类目属性Schema → 渲染动态表单
                ↓
        平台适配器注入额外字段
                ↓
        国别配置注入本地化字段
                ↓
        表单验证规则生成
                ↓
        提交 → 平台适配器校验 → 提交到平台
```

**动态表单字段类型**:
- 文本输入、数字输入、下拉选择、多选、开关、日期选择
- 图片上传、视频上传、富文本编辑器
- 动态数组（SKU变体、规格选项）
- 嵌套对象（地址、物流信息）

**表单验证规则**:
- 必填项校验
- 字符长度限制
- 数值范围限制
- 格式校验（手机号、邮箱、URL）
- 平台特定校验（类目必填、资质必填、实拍图必填等）

### 1.5 SKU/变体管理设计规范

基于四平台编辑页面调研，CBHunter支持两种SKU模式：

#### 1.5.1 两层规格模式（Shopee/TikTok/妙手）

```
规格一：颜色（红色/蓝色/绿色）
规格二：尺寸（小号/中号/大号）

生成SKU矩阵：
┌─────────┬─────────┬─────────┬─────────┐
│ 红色    │ 蓝色    │ 绿色    │         │
├─────────┼─────────┼─────────┼─────────┤
│ 小号    │ 红-小   │ 蓝-小   │ 绿-小   │
│ 中号    │ 红-中   │ 蓝-中   │ 绿-中   │
│ 大号    │ 红-大   │ 蓝-大   │ 绿-大   │
└─────────┴─────────┴─────────┴─────────┘
```

#### 1.5.2 三层SKU模式（TEMU全托管）

```
SPU（标准产品单元）
├── SKC 1（标准颜色单元 - 黑色）
│   ├── SKU 1（黑色/小号）
│   └── SKU 2（黑色/中号）
└── SKC 2（标准颜色单元 - 棕色）
    ├── SKU 3（棕色/小号）
    └── SKU 4（棕色/中号）
```

#### 1.5.3 SKU图片管理

参考妙手ERP的独立SKU图片管理区：
- 主图：商品级，最多9张
- SKU图片：SKU级别，支持拖拽排序
- 批量上传：支持Excel导入SKU图片映射

#### 1.5.4 SKU生成规则

参考妙手ERP的灵活规则：
- 默认规则：`货源ID + 规格1 + 规格2`
- 分隔符可配（下划线/横杠/空）
- 超限保护：超过100个SKU时提示而非自动删除

### 1.6 图片编辑器集成方案

#### 1.6.1 架构设计

```
ImageEditorComponent
├── 基础版（内置）
│   ├── 上传/预览
│   ├── 裁剪/旋转
│   ├── 尺寸修改
│   └── 水印添加
└── 增强版（可选集成易可图）
    ├── iframe嵌入 yiketu.com/photo/editor
    ├── 批量编辑（最多10张）
    ├── AI设计/智能抠图
    └── 主图模板库
```

#### 1.6.2 妙手ERP的启示

妙手ERP将图片编辑器作为独立iframe嵌入批量编辑弹窗：
- **解耦**：图片编辑器独立维护，不阻塞主表单
- **批量**：支持一次性编辑多张图片
- **功能丰富**：AI设计、智能抠图、图片翻译等高级功能
- **扩展**：可替换为自研编辑器或第三方服务

#### 1.6.3 CBHunter实现建议

**第一阶段（MVP）**：使用内置轻量级编辑器
- 支持：上传、预览、裁剪、旋转、水印
- 图片存储到CDN，返回URL

**第二阶段**：集成易可图或类似SaaS服务
- 支持批量编辑（10张/次）
- 支持AI设计、智能抠图等高级功能

**第三阶段**：根据用户需求自研完整编辑器
- 支持更多电商专用功能（主图模板、商品堆品、切图等）

### 1.7 批量编辑功能设计

参考妙手ERP的批量编辑弹窗，支持以下批量操作：

| 操作 | 说明 | 实现方式 |
|------|------|----------|
| 批量标题优化 | AI自动优化/关键词随机组合 | NLP API + 规则引擎 |
| 批量图片编辑 | 调用图片编辑器批量处理 | iframe嵌入 |
| 批量SKU修改 | 表格内直接编辑 | 动态表格组件 |
| 批量价格调整 | 公式计算（如统一加价10%） | 计算器组件 |
| 批量类目设置 | 类目选择器 | 下拉选择 |
| Excel粘贴支持 | 产品标题搜索支持逗号分隔 | 文本解析器 |

**批量编辑流程**：
```
1. 用户在列表页勾选多个商品
2. 点击"批量编辑"按钮
3. 打开批量编辑弹窗
4. 选择要编辑的字段（标题/图片/SKU/价格/类目/描述）
5. 在弹窗内编辑（支持表格内直接修改）
6. 点击"保存"批量提交
7. 显示编辑结果（成功/失败数量）
```
