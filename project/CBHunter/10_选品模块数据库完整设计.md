# 选品模块数据库完整设计

> **项目**: CBHunter — 多平台跨境电商卖家后台  
> **文档类型**: 数据库架构设计  
> **版本**: v2.0  
> **日期**: 2026-07-21  
> **核心原则**: 策略与数据源解耦、支持无限嵌套、预留D级策略扩展

---

## 一、数据库设计总览

```
┌─────────────────────────────────────────────────────────────┐
│                    PostgreSQL Database                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────┐ │
│  │ 策略配置表       │  │ 复合策略关联表   │  │ 数据源映射表 │ │
│  │ base_strategies │→│ composite_links │←│ ds_mapping  │ │
│  └─────────────────┘  └─────────────────┘  └─────────────┘ │
│         ↓                     ↓                  ↓          │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────┐ │
│  │ 策略参数表       │  │ 选品任务表       │  │ 商品样例快照│ │
│  │ strategy_params │  │ selection_jobs  │  │ product_snap│ │
│  └─────────────────┘  └─────────────────┘  └─────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**设计原则**：
- 所有策略配置独立存储，不依赖数据源是否存在
- 复合策略支持无限嵌套（自引用表结构）
- 数据源映射可热替换，不影响策略定义
- 选品结果持久化为快照，支持回溯和对比

## 二、核心表结构设计

### 2.1 原生策略表（base_strategies）

```sql
CREATE TABLE base_strategies (
    id BIGSERIAL PRIMARY KEY,
    strategy_id VARCHAR(20) NOT NULL UNIQUE,  -- 如 BS-01, BL-01
    name VARCHAR(100) NOT NULL,
    category VARCHAR(50) NOT NULL,            -- 爆款/蓝海/利润/竞争/趋势/合规
    description TEXT,
    data_feasibility CHAR(1) NOT NULL CHECK (data_feasibility IN ('A', 'B', 'C', 'D')),
    
    -- 适配平台（JSON数组）
    platforms JSONB NOT NULL DEFAULT '[]',
    
    -- 策略参数Schema（JSON Schema格式）
    parameters_schema JSONB NOT NULL DEFAULT '{}',
    
    -- 阈值配置Schema
    thresholds_schema JSONB NOT NULL DEFAULT '{}',
    
    -- 默认参数值
    default_parameters JSONB NOT NULL DEFAULT '{}',
    default_thresholds JSONB NOT NULL DEFAULT '{}',
    
    -- 数据源引用（可为空，D级策略为空）
    data_source_ref VARCHAR(50),
    
    -- 状态管理
    status VARCHAR(20) NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'inactive', 'deprecated')),
    is_active BOOLEAN NOT NULL DEFAULT true,
    
    -- 元数据
    version INTEGER NOT NULL DEFAULT 1,
    created_by BIGINT REFERENCES users(id),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    deleted_at TIMESTAMP WITH TIME ZONE
);

CREATE INDEX idx_base_strategies_category ON base_strategies(category);
CREATE INDEX idx_base_strategies_feasibility ON base_strategies(data_feasibility);
CREATE INDEX idx_base_strategies_status ON base_strategies(status);
```

**字段说明**：
- `strategy_id`: 全局唯一标识，格式为"大类代码-序号"（如 BS-01）
- `parameters_schema`: 定义该策略支持哪些可配置参数（JSON Schema格式）
- `data_source_ref`: 关联的数据源ID，D级策略为NULL
- `deleted_at`: 软删除，保留历史记录

### 2.2 策略参数实例表（strategy_parameters）

用于存储用户自定义的参数配置（覆盖默认值）：

```sql
CREATE TABLE strategy_parameters (
    id BIGSERIAL PRIMARY KEY,
    strategy_id VARCHAR(20) NOT NULL REFERENCES base_strategies(strategy_id),
    job_id BIGINT REFERENCES selection_jobs(id),  -- 可选，关联具体选品任务
    
    -- 参数键值对（灵活存储任意参数）
    param_key VARCHAR(100) NOT NULL,
    param_value JSONB NOT NULL,
    
    -- 是否使用默认值
    is_default BOOLEAN NOT NULL DEFAULT false,
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    
    UNIQUE(strategy_id, job_id, param_key)
);

CREATE INDEX idx_strategy_params_job ON strategy_parameters(job_id);
```

### 2.3 复合策略表（composite_strategies）

支持无限嵌套的自引用结构：

```sql
CREATE TABLE composite_strategies (
    id BIGSERIAL PRIMARY KEY,
    composite_id VARCHAR(50) NOT NULL UNIQUE,  -- 如 CS-001, CS-002
    name VARCHAR(100) NOT NULL,
    description TEXT,
    
    -- 运算类型
    operator_type VARCHAR(20) NOT NULL CHECK (operator_type IN (
        'intersection',  -- ∩ 交集
        'union',         -- ∪ 并集
        'difference',    -- − 差集
        'weighted'       -- Σ 加权
    )),
    
    -- 父节点ID（多级嵌套用）
    parent_composite_id VARCHAR(50) REFERENCES composite_strategies(composite_id),
    
    -- 权重（加权模式下使用，0-1）
    weight DECIMAL(3,2) NOT NULL DEFAULT 1.0 CHECK (weight >= 0 AND weight <= 1),
    
    -- 策略树JSON（存储完整嵌套结构，便于快速查询）
    strategy_tree JSONB NOT NULL,
    
    -- 版本管理
    version INTEGER NOT NULL DEFAULT 1,
    is_current_version BOOLEAN NOT NULL DEFAULT true,
    
    -- 状态
    status VARCHAR(20) NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'draft', 'archived')),
    is_shared BOOLEAN NOT NULL DEFAULT false,
    shared_with_group_id BIGINT REFERENCES user_groups(id),
    
    -- 元数据
    created_by BIGINT REFERENCES users(id),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    deleted_at TIMESTAMP WITH TIME ZONE
);

CREATE INDEX idx_composite_parent ON composite_strategies(parent_composite_id);
CREATE INDEX idx_composite_status ON composite_strategies(status);
CREATE INDEX idx_composite_created ON composite_strategies(created_at);
```

**strategy_tree 结构示例**：

```json
{
  "type": "composite",
  "operator": "intersection",
  "children": [
    {
      "type": "base",
      "ref_id": "BS-01",
      "weight": 0.3,
      "config": {"top_n": 50, "time_window": "7d"}
    },
    {
      "type": "composite",
      "ref_id": "CS-002",
      "weight": 0.7,
      "children": [...]
    }
  ]
}
```

### 2.4 复合策略关联表（composite_links）

记录复合策略中每个子节点的详细配置：

```sql
CREATE TABLE composite_links (
    id BIGSERIAL PRIMARY KEY,
    composite_id VARCHAR(50) NOT NULL REFERENCES composite_strategies(composite_id),
    
    -- 子节点类型
    child_type VARCHAR(10) NOT NULL CHECK (child_type IN ('base', 'composite')),
    
    -- 引用ID
    ref_id VARCHAR(50) NOT NULL,  -- 原生策略ID或复合策略ID
    
    -- 运算类型（节点间的关系）
    link_operator VARCHAR(20) NOT NULL CHECK (link_operator IN (
        'intersection', 'union', 'difference', 'weighted'
    )),
    
    -- 权重（加权模式）
    weight DECIMAL(3,2) DEFAULT 1.0,
    
    -- 排序（同一层级内的顺序）
    sort_order INTEGER NOT NULL DEFAULT 0,
    
    -- 额外配置
    extra_config JSONB NOT NULL DEFAULT '{}',
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    
    UNIQUE(composite_id, ref_id, link_operator, sort_order)
);

CREATE INDEX idx_composite_links_composite ON composite_links(composite_id);
```

### 2.5 数据源注册表（data_sources）

```sql
CREATE TABLE data_sources (
    id BIGSERIAL PRIMARY KEY,
    source_id VARCHAR(50) NOT NULL UNIQUE,  -- 如 DS-TIKTOK-BESTSELLER
    name VARCHAR(100) NOT NULL,
    description TEXT,
    
    -- 数据源类型
    source_type VARCHAR(20) NOT NULL CHECK (source_type IN (
        'crawler',   -- A级：爬虫
        'api',       -- B级：API
        'import',    -- C级：导入
        'manual',    -- C级：手动
        'none'       -- D级：无
    )),
    
    -- 适配平台
    platforms JSONB NOT NULL DEFAULT '[]',
    
    -- 采集配置
    url_pattern VARCHAR(500),
    api_endpoint VARCHAR(500),
    auth_type VARCHAR(20),  -- none/basic/oauth2/api_key
    auth_config JSONB,
    
    -- 刷新策略
    refresh_frequency VARCHAR(20) NOT NULL DEFAULT 'daily' CHECK (refresh_frequency IN (
        'hourly', 'daily', 'weekly', 'on_demand'
    )),
    max_items_per_fetch INTEGER DEFAULT 1000,
    
    -- 字段映射规则（原始字段 → 标准字段）
    field_mapping JSONB NOT NULL DEFAULT '{}',
    
    -- 健康状态
    health_status VARCHAR(20) NOT NULL DEFAULT 'healthy' CHECK (health_status IN (
        'healthy', 'degraded', 'down', 'deprecated'
    )),
    last_success_at TIMESTAMP WITH TIME ZONE,
    last_error_at TIMESTAMP WITH TIME ZONE,
    error_message TEXT,
    
    -- 限流配置
    rate_limit_per_minute INTEGER DEFAULT 60,
    rate_limit_per_day INTEGER DEFAULT 10000,
    
    -- 元数据
    created_by BIGINT REFERENCES users(id),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    deleted_at TIMESTAMP WITH TIME ZONE
);

CREATE INDEX idx_data_sources_type ON data_sources(source_type);
CREATE INDEX idx_data_sources_health ON data_sources(health_status);
```

### 2.7 选品任务表（selection_jobs）

每次运行选品策略生成一个任务记录：

```sql
CREATE TABLE selection_jobs (
    id BIGSERIAL PRIMARY KEY,
    job_uuid VARCHAR(36) NOT NULL UNIQUE,  -- 任务UUID
    
    -- 关联策略
    composite_strategy_id VARCHAR(50) REFERENCES composite_strategies(composite_id),
    strategy_snapshot JSONB NOT NULL,  -- 策略执行时的快照（防止后续修改影响历史）
    
    -- 任务状态
    status VARCHAR(20) NOT NULL DEFAULT 'pending' CHECK (status IN (
        'pending', 'running', 'completed', 'failed', 'cancelled'
    )),
    
    -- 执行信息
    started_at TIMESTAMP WITH TIME ZONE,
    completed_at TIMESTAMP WITH TIME ZONE,
    duration_ms INTEGER,
    
    -- 结果统计
    total_matched INTEGER DEFAULT 0,
    total_processed INTEGER DEFAULT 0,
    data_missing_count INTEGER DEFAULT 0,
    
    -- 错误信息
    error_message TEXT,
    
    -- 触发方式
    trigger_type VARCHAR(20) NOT NULL DEFAULT 'manual' CHECK (trigger_type IN (
        'manual', 'scheduled', 'api'
    )),
    
    -- 元数据
    created_by BIGINT REFERENCES users(id),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX idx_selection_jobs_status ON selection_jobs(status);
CREATE INDEX idx_selection_jobs_created ON selection_jobs(created_at);
CREATE INDEX idx_selection_jobs_strategy ON selection_jobs(composite_strategy_id);
```

```sql
-- 选品结果可直接生成SKU定价草稿
INSERT INTO sku_pricing_drafts (
    product_id, sku_id, cost_cny, target_margin, suggested_price
)
SELECT 
    p.id,
    s.id,
    bp.cost_cny,
    bp.target_margin,
    ROUND(bp.cost_cny * (1 + bp.target_margin), 2)

JOIN skus s ON p.id = s.product_id
JOIN benchmark_products bp ON cp.id = bp.candidate_product_id;
```

## 八、数据库维护脚本

### 8.1 清理过期数据

```sql
-- 清理90天前的已完成选品任务
DELETE FROM selection_jobs 
WHERE status = 'completed' 
AND completed_at < NOW() - INTERVAL '90 days';

-- 清理关联的候选商品
DELETE FROM candidate_products 
WHERE job_id NOT IN (SELECT id FROM selection_jobs);
```

### 8.2 数据源健康检查

```sql
-- 检查异常数据源
SELECT ds.source_id, ds.name, ds.health_status, ds.last_error_at
FROM data_sources ds
WHERE ds.health_status != 'healthy'
OR ds.last_error_at > NOW() - INTERVAL '1 hour';
```

## 十一、数据库维护脚本（v2 扩展）

### 11.1 策略胜率自动更新

```sql
-- 每次选品任务完成后，自动更新策略胜率统计
CREATE OR REPLACE FUNCTION update_strategy_win_rate()
RETURNS TRIGGER AS $$
BEGIN
    IF NEW.status = 'completed' THEN
        -- 更新复合策略胜率
        INSERT INTO strategy_win_rates (
            composite_strategy_id, period_start, period_end,
            execution_count, avg_matched_products, products_with_sales, hit_rate
        )
        VALUES (
            NEW.composite_strategy_id,
            DATE_TRUNC('week', NEW.created_at)::date,
            DATE_TRUNC('week', NEW.created_at)::date + INTERVAL '6 days',
            1,
            NEW.total_matched,
            (SELECT COUNT(*) FROM candidate_products 
             WHERE job_id = NEW.id AND sales_30d > 0),
            ROUND(
                (SELECT COUNT(*) FILTER (WHERE sales_30d > 0)::numeric 
                 / NULLIF(COUNT(*), 0) * 100)
            , 2)
            FROM candidate_products WHERE job_id = NEW.id)
        ON CONFLICT (composite_strategy_id, period_start, period_end)
        DO UPDATE SET
            execution_count = strategy_win_rates.execution_count + 1,
            avg_matched_products = (
                SELECT ROUND(
                    AVG(candidate_products.total_matched)::numeric, 2)
                FROM candidate_products 
                WHERE candidate_products.job_id = NEW.id
            ),
            updated_at = NOW()
        ;
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_selection_job_completed
AFTER INSERT OR UPDATE OF status ON selection_jobs
FOR EACH ROW WHEN (NEW.status = 'completed')
EXECUTE FUNCTION update_strategy_win_rate();
```
