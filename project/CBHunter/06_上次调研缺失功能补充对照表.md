## 一、妙手ERP产品图片编辑功能（核心缺失）

### 1.1 缺失描述

首次调研仅覆盖了妙手ERP的采集箱、批量发布、字段映射等通用功能，**未覆盖产品编辑弹窗中的图片编辑功能**。该功能是卖家日常运营最高频的操作之一，直接影响CBHunter商品编辑模块的设计质量。

### 1.2 补充调研内容

**调研方式**: Chrome DevTools MCP 只读抓取妙手ERP产品列表页内嵌弹窗

**发现的功能架构**: 双层弹窗模式

```
第一层：批量编辑弹窗（el-dialog）
├── 标签页导航：标题管理 | 关键词 | 属性 | 图片 | 描述
├── 销售属性/SKU信息区
│   ├── 多规格开关（一键切换单规格/多规格模式）
│   ├── 规格一输入
│   └── 规格二输入
├── 产品图片区
│   ├── 主图管理
│   └── SKU图片管理
└── 底部操作按钮：批量编辑 | 保存
    └── 第二层：易可图图片编辑器（iframe）
        └── https://www.yiketu.com/photo/editor
```

### 1.3 关键功能点

| 功能点 | 妙手ERP实现 | CBHunter借鉴 |
|--------|------------|-------------|
| **编辑形式** | 弹窗式（Modal） | ✅ 采用相同设计 |
| **SKU管理** | 多规格开关 + 规格一/规格二两行定义 | ✅ 两层规格模式 |
| **SKU图片** | 独立SKU图片管理区 | ✅ SKU级别图片管理 |
| **图片编辑器** | iframe嵌入易可图在线编辑器 | ✅ 组件化集成方案 |
| **批量编辑** | 弹窗内直接编辑多个商品 | ✅ 批量编辑弹窗 |
| **Excel粘贴** | 产品标题搜索支持逗号分隔和Excel复制粘贴 | ✅ 采用相同设计 |

### 1.4 易可图图片编辑器功能清单

| 功能模块 | 具体功能 |
|----------|----------|
| **基础调整** | 调整、裁剪旋转、修改尺寸、图片校正 |
| **添加元素** | 文字、标记、形状线条、马赛克、标注笔、放大镜 |
| **主图模板** | 电商主图模板库 |
| **色彩处理** | 色彩调整 |
| **水印** | 水印添加 |
| **素材库** | 电商素材、边框、组件 |
| **AI功能** | AI设计、智能抠图、图片变清晰、商品堆品、logo检测 |
| **高级工具** | 消除笔、图片翻译、拼图、切图、无损放大、POD设计、单位转化、商品标尺、涂鸦 |

### 1.5 对CBHunter设计的影响

1. **编辑入口改为弹窗式**：从独立页面编辑改为弹窗式编辑，保持列表页上下文
2. **SKU管理支持两层规格**：规格一 + 规格二，参考妙手ERP的多规格开关设计
3. **图片编辑器解耦**：通过iframe嵌入，支持批量编辑和后期替换
4. **批量编辑前置**：在采集箱和编辑阶段都提供批量操作能力

## 三、CBHunter开发指导意见总结

### 3.1 核心架构决策

基于四平台编辑页面深度调研，CBHunter采用**"统一底座 + 平台适配器 + 弹窗式编辑"**架构。

| 决策点 | 推荐方案 | 理由 |
|--------|----------|------|
| 编辑形式 | 弹窗式（妙手风格） | 批量操作友好，上下文保持 |
| SKU管理 | 两层+三层混合 | 兼容普通平台和TEMU全托管 |
| 图片编辑 | 组件化iframe嵌入 | 解耦，可替换 |
| 动态表单 | Schema驱动 | 类目属性动态加载 |
| 字段存储 | JSONB扩展 | 平台差异大，固定表难以覆盖 |
| 定价引擎 | 统一CNY存储 + 国别格式化 | Temu供货价/Shopee零售价都能表达 |
| 批量编辑 | 弹窗内表格编辑 | 参考妙手ERP效率最高 |

### 3.2 开发优先级

| 阶段 | 周期 | 核心任务 |
|------|------|----------|
| P0 | 4周 | 统一商品数据模型、基础编辑弹窗、SKU管理（简单模式）、平台适配器框架 |
| P1 | 4周 | 动态表单引擎、批量编辑功能、图片编辑器集成（基础版）、TEMU三层SKU模式 |
| P2 | 4周 | 易可图式增强图片编辑器、AI标题优化、平台特有功能集成、Excel批量导入/导出 |
