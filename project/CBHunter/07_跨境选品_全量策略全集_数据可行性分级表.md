## 一、数据可行性分级体系（v2 增强版）

所有选品策略统一采用四级数据可行性分级，确保无数据策略不丢失、可配置、可迭代。

| 级别 | 定义 | 说明 | 当前落地方式 | 未来升级路径 |
|------|------|------|-------------|-------------|
| **A级** | 公开展示数据 | 无需登录、无需授权，浏览器可见即可获取 | Chrome DevTools MCP只读爬取 / 爬虫 | 自动化定时采集、API对接 |
| **B级** | 店铺授权后数据 | 需卖家账号授权，通过官方API或卖家后台同步 | 妙手ERP授权同步 / 官方API | 直连平台开放平台API |
| **C级** | 人工/表格补录 | 无自动数据源，需用户手动导入或批量上传 | Excel/CSV导入、表单填写 | 第三方SaaS数据购买、OCR识别 |
| **D级** | 纯策略预留 | 商业逻辑成立但当前无任何数据源 | 仅作为策略配置项存在 | 待数据源开发后激活 |

### v2 新增：数据获取精准定级体系

每条策略除ABCD四级外，还需标注以下五维数据资产信息：

| 维度 | 枚举值 | 说明 |
|------|--------|------|
| **精准渠道来源** | 公域页面 / 搜索建议 / 类目树 / 商品详情页 / 店铺页 / 直播/短视频页 / 平台官方API / 卖家后台 / 妙手ERP同步 / 第三方工具 / 搜索引擎 / RSS / 人工导入 / 预留 | 数据的具体来源位置 |
| **获取难易度** | 极易 / 简单 / 中等 / 困难 / 无法获取 | 从技术实现角度评估采集难度 |
| **更新频率** | 实时 / 日更 / 周更 / 月更 / 静态人工 / 事件触发 | 数据刷新周期 |
| **数据成本** | 免费 / 低成本 / 付费 / 高成本 | 获取该数据所需费用 |
| **落地可行性** | 短期可落地(1-2周) / 中期可落地(1-3月) / 长期规划(3-6月) / 预留迭代 | 工程实现周期评估 |

**关键设计原则**：
- A/B/C/D四级策略**同等入库、同等配置、同等组合**
- 无数据的D级策略不影响复合策略运算，运算时标记为"数据缺失"
- 后续数据源接入后，策略从D级升级为A/B/C级，**无需修改核心引擎**
- 每条策略独立标注适配平台、适用场景、当前状态、数据资产五维信息

## 三、复合策略运算规则

### 3.1 基础运算类型

| 运算类型 | 符号 | 说明 | 示例 |
|----------|------|------|------|
| 交集 | ∩ | 同时满足多个策略条件 | BS-01 ∩ BL-01 = 榜单Top50且低竞争的类目 |
| 并集 | ∪ | 满足任一策略条件即可 | BS-01 ∪ TR-02 = 爆款或趋势上升商品 |
| 差集 | \ | 排除满足特定条件的商品 | BS-01 \ CP-01 = 爆款但非高集中度类目 |
| 加权评分 | Σ(w×s) | 多策略加权求和排序 | 0.3×BS-01 + 0.2×BL-01 + 0.5×PF-01 |

### 3.2 复合策略嵌套结构

```
复合策略根节点
├── 原生策略 BS-01 [权重0.3]
├── 复合策略子节点1 [权重0.2]
│   ├── 原生策略 BL-01 [权重0.6]
│   └── 原生策略 CP-02 [权重0.4]
├── 复合策略子节点2 [权重0.5]
│   ├── 复合策略子节点2-1 [权重0.7]
│   │   ├── 原生策略 PF-01 [权重0.5]
│   │   └── 原生策略 PF-02 [权重0.5]
│   └── 原生策略 RC-01 [权重0.3]
└── 运算模式：交集/并集/差集/加权
```

**核心特性**：
- **无限嵌套**：复合策略可作为子节点继续组合
- **权重可调**：每个节点支持独立权重（0-1）
- **运算切换**：节点间可切换交集/并集/差集
- **永久保存**：所有策略组合可保存为独立策略实体
- **复用迭代**：已保存策略可再次作为组件使用

### 3.3 复合策略另存为新策略

当用户完成一组策略组合后，可选择：
1. **另存为一级复合策略**：当前组合命名为新策略，存入策略库
2. **另存为多级嵌套策略**：保留嵌套结构，整体存入策略库
3. **导出为模板**：供其他用户或未来会话复用

## 四、策略与数据源解耦设计

### 4.1 解耦架构

```
┌─────────────────────────────────────────────────┐
│              策略配置层（Strategy Layer）          │
│  原生策略库 | 复合策略库 | 权重配置 | 阈值设置     │
│  ↓ 完全独立，不依赖数据源是否存在                 │
├─────────────────────────────────────────────────┤
│              运算引擎层（Engine Layer）            │
│  交集/并集/差集/加权运算 | 结果排序 | 去重         │
│  ↓ 仅读取策略配置，不直接访问数据                  │
├─────────────────────────────────────────────────┤
│              数据源映射层（Data Source Layer）      │
│  策略ID → 数据源ID | 字段映射 | 采集方式 | 频率    │
│  ↓ 数据源可独立开发、替换、升级                    │
├─────────────────────────────────────────────────┤
│              数据采集层（Collection Layer）        │
│  A级：爬虫 | B级：API | C级：导入 | D级：无       │
└─────────────────────────────────────────────────┘
```

### 4.2 无数据策略处理

当策略处于D级（无数据源）时：
- 策略**正常显示、正常配置、正常组合**
- 运算时该策略**标记为"数据缺失"**，不参与实际筛选
- 用户可看到"因数据缺失，以下策略未生效"提示
- 后续数据源接入后，策略自动升级为A/B/C级，**无需重新配置**

### 4.3 数据源升级路径

| 当前级别 | 升级目标 | 实现方式 | 对策略层影响 |
|----------|----------|----------|-------------|
| D级 | C级 | 用户手动导入数据 | 仅更新数据源映射，策略不变 |
| D级 | B级 | 开通卖家授权+API对接 | 仅更新数据源映射，策略不变 |
| D级 | A级 | 开发爬虫采集公域数据 | 仅更新数据源映射，策略不变 |
| C级 | B级 | 从人工导入升级为API同步 | 仅更新数据源映射，策略不变 |
| B级 | A级 | 从授权数据升级为公域数据 | 仅更新数据源映射，策略不变 |

## 七、策略组合运算示例

### 示例1：蓝海爆款组合

**目标**：找到TikTok菲律宾站近7天榜单Top50、且搜索量上升、竞争度低的商品

**策略组合**：
```
BS-01 (榜单Top50) ∩ TR-02 (搜索趋势上升) ∩ BL-01 (低竞争高需求)
```

**运算结果**：
- 输入：BS-01返回50个商品，TR-02返回200个商品，BL-01返回300个商品
- 交集运算后：约15个商品
- 输出：标杆样例商品列表（仅作为选品参考方向）

### 示例2：利润导向的爆款筛选

**目标**：Shopee多站点热销商品中，毛利率≥30%、运费占比≤10%的品类

**策略组合**：
```
BS-01 (榜单Top100) ∩ PF-01 (毛利率≥30%) ∩ PF-02 (运费占比≤10%)
```

**运算结果**：
- 输入：BS-01返回100个商品，PF-01需用户导入成本数据，PF-02需物流估算
- 加权评分：0.4×BS-01 + 0.3×PF-01 + 0.3×PF-02
- 输出：按综合得分排序的商品候选池

### 示例3：多级嵌套复合策略

**目标**：合规前提下，寻找跨平台蓝海爆款

**策略组合**：
```
[(BS-01 ∩ BL-03) ∩ (RC-01 ∪ RC-02)] ∩ PF-01
```

**嵌套结构**：
```
根节点（交集）
├── 子节点1（交集）
│   ├── BS-01 (榜单Top50)
│   └── BL-03 (跨站蓝海迁移)
├── 子节点2（并集）
│   ├── RC-01 (禁限售规避)
│   └── RC-02 (知识产权风险)
└── PF-01 (毛利率≥30%)
```
