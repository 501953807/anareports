## 2. 多平台上架工作流

### 2.1 商品采集流程

**采集方式**（7种）：
1. **链接采集** — 粘贴产品链接或ID，支持自定义新标题（`链接$$新标题` 或 `链接||新标题`）
2. **1688跨境热卖** — 直接从1688跨境频道采集
3. **导入采集** — 批量导入采集
4. **关键词采集** — 按关键词搜索采集
5. **整店采集** — 采集整个店铺的商品
6. **插件采集** — 浏览器插件辅助采集
7. **店铺互采** — 店铺间互相采集

**采集设置弹窗**（核心字段映射配置）：
- 平台认领配置：Shopee / TikTok 分平台独立配置
- 通用配置：产品状况（全新/二手）、是否预购、发货天数
- 售价预设：跨境全球 / 本地分组
- 库存设置：本土仓库存默认值
- 尺码表：尺码图表 / 尺码模板
- SKU设置：主货号生成规则、平台SKU生成规则（货源ID+规格1+规格2，分隔符可配）
- 图片设置：翻译、编辑、导出、图片补全（将描述图补充到产品图片中）
- 标题优化：AI自动优化（付费）/ 关键词随机组合 / 暂不优化
- 描述设置：简易描述 / 详细描述 / 自定义描述
- 过滤来源品牌信息

### 2.2 编辑/发布流程

**采集箱页面**（以Shopee采集箱为例，3832个已发布商品）：

**筛选条件**：
- 关联货源（全部关联货源）
- 货源店铺
- 店铺（已发布店铺）
- 货源价格范围
- 是否标记
- 产品标题（多标题匹配，支持Excel复制粘贴）
- 货源ID（支持Excel复制粘贴）

**批量操作按钮**：
- 批量发布
- 设置类目
- 设置售价
- 批量编辑
- 引用模板
- 分组
- 创建产品

**状态标签**：全部(3832) / 未发布(0) / 定时发布(0) / 已发布(3832)

**列表字段**：
| 字段 | 说明 |
|------|------|
| 产品信息 | 商品图片+标题 |
| 分组 | 产品分组 |

### 2.3 批量操作设计

妙手的批量操作集中在采集箱场景：
- **批量发布**：选中多个商品一次性发布到目标平台
- **批量编辑**：对多个商品进行统一修改
- **设置类目**：批量指定发布类目
- **设置售价**：批量调整售价
- **引用模板**：批量套用定价/产品模板
- **分组**：批量归类

### 2.4 平台适配机制

- 采集箱按平台隔离：每个平台有独立的采集箱（如 `shopee/collect_box/items`）
- 采集设置按平台独立配置：Shopee和TikTok各自有不同的预购设置、售价预设、SKU规则
- 公告消息按平台筛选：Shopee / TikTok / 其他
- 直播培训内容按平台区分：Temu、Shopee、TikTok、Ozon等

## 3. 字段映射策略

### 3.1 统一字段 vs 平台专属字段

**统一字段**（跨平台通用）：
- 产品信息（标题、图片、描述）
- 货源价格
- 库存
- 重量
- 备注
- 分组

**平台专属字段**（采集设置弹窗中体现）：
- **Shopee**：预购设置、发货天数（跨境1-10天，本地4-30天）、尺码表、NCC认证、Anatel认证（Model Name / Registration ID / 生产商）
- **TikTok**：独立的平台认领配置

### 3.2 分类映射逻辑

- 发布类目通过"预发布类目"字段管理
- 类目设置是发布前的必要步骤（采集箱中有"设置类目"批量按钮）
- 标题优化支持AI自动优化和关键词随机组合
- 描述设置支持简易描述/详细描述/自定义描述的灵活组合

### 3.3 SKU 同步策略

**SKU生成规则**（Shopee采集设置）：
- 主货号：按内置规则生成 / 自定义 / 留空
- 平台SKU：按内置规则生成 / 自定义 / 留空
- 默认规则：`货源ID + 规格1 + 规格2`，分隔符可选（如下划线），中文处理可选
- 超限保护：来源超过100个SKU时自动删除部分规格选项；SKU名称/选项超字符限制时自动截断

**库存同步**：
- 本土仓库存默认为0（未开启配置则按采集所得库存自动平均分配至各店铺仓库）

### 3.4 定价转换规则

- 售价预设支持"跨境全球"和"本地分组"两种模式
- 定价模板和产品模板可被批量引用
- 货源价格与售价分开显示，便于利润计算

## 5. CBHunter 可借鉴的设计模式

### 5.1 值得采用的架构模式

1. **按平台隔离采集箱**：每个销售平台独立采集箱，避免不同平台字段冲突
2. **采集设置分平台独立配置**：Shopee/TikTok等平台各自维护自己的预处理规则
3. **批量操作前置**：在采集箱阶段就提供批量发布/编辑/设置类目/设置售价，减少后续修改成本
4. **货源-售价分离展示**：列表中同时展示货源价格和售价，直观呈现利润空间
5. **状态机管理**：未发布/定时发布/已发布三态清晰

### 5.2 值得学习的 UX 设计

1. **采集设置弹窗**：将复杂的字段映射、SKU生成、标题优化、描述处理集中在一个可配置的弹窗中，支持保存为模板
2. **Excel粘贴支持**：产品标题和货源ID搜索支持逗号分隔和Excel复制粘贴，提升批量操作效率
3. **分组系统**：产品分组功能，支持搜索分组和管理分组
4. **AI助理悬浮入口**：降低用户寻求帮助的成本
5. **平台公告按类别筛选**：Shopee/TikTok/其他分类，帮助用户快速定位相关信息

### 5.3 需要避免的陷阱

1. **页面重定向不稳定**：MCP导航时页面频繁被其他Tab抢占，需要在架构上考虑页面状态管理
2. **Temu图片加载问题**：境外服务器图片在国内访问慢，需要提前做好图片缓存或CDN策略
3. **SKU超限处理**：来源超过100个SKU时自动删除部分规格，需要设计合理的保留策略
4. **品牌信息过滤**：采集时默认过滤来源品牌信息，但需要给用户提供开关
5. **多标题匹配**：产品标题搜索支持多种匹配模式，需要明确各模式的语义差异
