## 六、Shopee 平台特色功能

### 6.1 Shopee标准商品 (Standard Products)

**概念**:
- Shopee平台统一管理的商品池
- 提高商品曝光和流量
- 支持跨店铺比价和竞争

**状态流转**:
```
可能匹配标准商品 → 符合标准商品 → 已链接标准商品 → Shopee标准商品
```

**SSP编号**:
- 标准商品唯一标识
- 格式: SSP400012702092126
- 用于Hot Listing和标准商品管理

**操作**:
- 链接标准商品
- 实时预览
- Check Detail / Check Details
- Check More

### 6.2 Hot Listing 计划

**概念**:
- Shopee的热门商品推广计划
- 帮助商品获得Buybox曝光
- 与标准商品系统联动

**功能**:
- Hot Listing Setting
- Buybox竞争分析
- 卖家数量提示 ("93 Sellers Linked And Get Traffic")
- 流量提升建议

**限制**:
- 每个版位可在每4小时后再次使用
- 置顶推广版位有限 (当前店铺: 所有5个版位已被使用)

### 6.3 联盟营销 (Affiliate Program)

**功能**:
- 设置联盟营销佣金
- 商品列表显示佣金金额
- 独立仪表盘: `https://seller.shopee.cn/portal/web-seller-affiliate/dashboard`

**佣金显示**:
- 商品列表显示佣金范围: ₱612 - ₱688
- 独立商品显示具体佣金

### 6.4 新品流量扶持

**功能**:
- "New Item (90day) Traffic Boosting"
- 90天新品流量提升计划
- 自动识别新品并给予流量倾斜

### 6.5 Smart Diagnosis

**功能**:
- 商品健康度诊断
- 内容问题提示 ("1 Content Issue To Fix")
- Optimise 按钮
- Bidding Eligible 资格判断
- Join 按钮加入竞价

### 6.6 Advance Fulfillment

**概念**:
- Shopee的预发货库存系统
- Available Stock: 已发货且可用的库存
- In-transit Advance Stock: 已发货但在途的库存

### 6.7 全托管服务

**入口**: 页面顶部 "全托管服务" 按钮
**功能**: Shopee全托管模式的服务入口

## 九、研究总结

### 9.1 关键发现

1. **标准商品系统是Shopee核心差异化功能**
   - 与Hot Listing深度绑定
   - 影响商品曝光和流量
   - 需要专门的产品经理理解

2. **类目选择是添加商品的关键门槛**
   - 必须先选类目才能继续
   - 不同类目有不同字段要求
   - 类目影响标准商品匹配

3. **营销工具丰富但复杂**
   - 7种优惠券类型
   - 折扣活动/Flash Sale/联盟营销
   - 需要清晰的UI引导

4. **订单处理注重效率**
   - 批量出货功能
   - 多物流渠道支持
   - 退货退款流程标准化

5. **财务模块有安全限制**
   - 需要额外密码验证
   - 涉及真实资金流动
   - CBHunter应避免直接操作

### 9.2 数据完整性评估

**已完整采集**:
- 商品列表页: ✅ 完整
- 添加商品流程: ⚠️ 仅基本信息，销售/运费/其他需类目选择
- 营销中心: ✅ 完整 (折扣/秒杀/优惠券)
- 订单模块: ✅ 完整
- 财务模块: ❌ 需额外认证
- 类目系统: ⚠️ 需进一步探索类目树

**建议后续研究**:
1. 完成类目树的完整采集
2. 解决财务模块认证问题
3. 深入探索销售资料/运费/其他字段
4. 研究Shopee广告模块
5. 分析跨境活动报名流程

### 9.3 对CBHunter产品的价值

本研究为CBHunter提供了:
- **字段级数据规范**: 可直接用于数据库设计和API对接
- **UI/UX参考**: 基于真实Shopee卖家中心的交互设计
