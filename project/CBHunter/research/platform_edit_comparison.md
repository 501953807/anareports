## 一、四平台产品编辑架构对比

### 1.1 编辑模式总览

| 维度 | Shopee | TikTok Shop | TEMU | 妙手ERP |
|------|--------|-------------|------|---------|
| **编辑入口** | 商品列表→编辑 | 商品列表→编辑 | 商品列表→编辑 | 批量编辑弹窗/单产品编辑 |
| **编辑形式** | 独立页面（SPA） | 独立页面（SPA） | 独立页面（多步骤） | 弹窗（Modal）+ iframe子页面 |

## 三、妙手ERP产品编辑弹窗深度分析

### 3.1 弹窗架构

妙手ERP采用**双层弹窗架构**：

```
第一层：批量编辑弹窗（el-dialog）
├── 标签页导航：标题管理 | 关键词 | 属性 | 图片 | 描述
├── 销售属性/SKU信息区
│   ├── 多规格开关
│   ├── 规格一输入
│   └── 规格二输入
├── 产品图片区
│   ├── 主图管理
│   └── SKU图片管理
└── 底部操作按钮：批量编辑 | 保存
    └── 第二层：易可图图片编辑器（iframe）
        └── https://www.yiketu.com/photo/editor
```

## 四、CBHunter产品编辑模块开发指导意见

### 4.1 核心架构建议：统一底座 + 弹窗式编辑（推荐）

基于四平台调研，建议CBHunter采用**妙手ERP的双层弹窗架构**作为基础，原因如下：

**优势**：
1. **批量操作友好**：弹窗内可直接编辑多个商品，无需跳转页面
2. **上下文保持**：用户不离开列表页，操作效率高
3. **模块化扩展**：图片编辑器可作为独立iframe组件嵌入
4. **平台适配灵活**：各平台适配器可独立控制弹窗内的字段显示

**架构**：
```
产品列表页
└── 编辑弹窗（ElDialog / Ant Design Modal）
    ├── 左侧：步骤导航（Shopee风格）或 标签页（妙手风格）
    ├── 右侧：表单内容区
    │   ├── 基本信息标签页
    │   ├── 销售资料标签页（含SKU/变体管理）
    │   ├── 图片管理标签页（含易可图式编辑器）
    │   ├── 物流设置标签页
    │   └── 其他属性标签页
    └── 底部操作栏：保存草稿 | 创建/更新 | 发布
        └── iframe: 图片编辑器组件（可配置接入易可图或自研）
```

### 4.2 字段设计原则

#### 4.2.1 分层存储模型

```
Product（商品主表）
├── standard_fields（标准字段，JSONB）
│   ├── title: string
│   ├── description: text
│   ├── images: array（主图+附图）
│   ├── video_url: string
│   ├── category_path: string
│   └── brand: string
├── sku_table（SKU子表）
│   ├── sku_id: string（平台分配）
│   ├── seller_sku_code: string（商家货号）
│   ├── variant_attrs: jsonb（规格属性）
│   ├── price_cny: decimal
│   ├── stock_qty: integer
│   ├── dimensions: jsonb（长宽高）
│   ├── weight_g: integer
│   ├── images: array（SKU图片）
│   └── status: enum
├── platform_extension（平台扩展字段，JSONB）
│   ├── platform_code: shopee/tiktok temu
│   ├── ssp_id / sps_score / spu_id / skc_id
│   ├── supply_price_cny / retail_price_compliance
│   ├── warehouse_group / stocking_logic
│   └── custom_fields: jsonb
└── country_config（国别配置）
    ├── currency_code
    ├── tax_rate
    └── logistics_channels: jsonb
```

#### 4.2.2 动态表单引擎

参考Shopee的"选择类目后动态加载字段"机制：

```
类目选择 → 加载类目属性Schema → 渲染动态表单
                ↓
        平台适配器注入额外字段
                ↓
        国别配置注入本地化字段
                ↓
        表单验证规则生成
                ↓
        提交 → 平台适配器校验 → 提交到平台
```

**关键实现点**：
- Schema驱动：每个类目的属性定义存储为JSON Schema
- 平台适配器注入：Shopee注入SSP/Hot Listing字段，TEMU注入SPU/SKC/供货价字段
- 国别配置注入：货币、税费、物流渠道按站点动态切换

### 4.3 SKU/变体管理设计

#### 4.3.1 支持两种模式

1. **简单模式（妙手风格）**：
   - 规格一、规格二两行定义
   - 适合大多数平台（Shopee/TikTok/妙手）

2. **三层模式（TEMU风格）**：
   - SPU → SKC → SKU
   - 适合全托管模式
   - CBHunter内部统一用父级商品+变体组+SKU三层结构

#### 4.3.2 SKU图片管理

参考妙手ERP的独立SKU图片管理区：
- 主图：商品级，最多9张
- SKU图片：SKU级别，支持拖拽排序
- 批量上传：支持Excel导入SKU图片映射

#### 4.3.3 SKU生成规则

参考妙手ERP的灵活规则：
- 默认规则：`货源ID + 规格1 + 规格2`
- 分隔符可配（下划线/横杠/空）
- 超限保护：超过100个SKU时提示而非自动删除

### 4.4 图片编辑器集成方案

#### 4.4.1 推荐方案：组件化集成

```
ImageEditorComponent
├── 基础版（内置）
│   ├── 上传/预览
│   ├── 裁剪/旋转
│   ├── 尺寸修改
│   └── 水印添加
└── 增强版（可选集成易可图）
    ├── iframe嵌入 yiketu.com/photo/editor
    ├── 批量编辑（最多10张）
    ├── AI设计/智能抠图
    └── 主图模板
```

#### 4.4.2 妙手ERP的启示

妙手ERP将图片编辑器作为独立iframe嵌入弹窗，这个设计值得借鉴：
- **解耦**：图片编辑器独立维护，不阻塞主表单
- **扩展**：可替换为自研编辑器或第三方服务
- **批量**：支持一次性编辑多张图片

**CBHunter实现建议**：
1. 初期使用自研轻量级图片编辑器（裁剪/旋转/水印）
2. 中期考虑集成易可图或类似SaaS服务
3. 后期根据用户需求决定是否自研完整编辑器

### 4.5 批量编辑功能设计

参考妙手ERP的批量编辑弹窗：

| 功能 | 实现方式 |
|------|----------|
| 批量标题优化 | AI自动优化/关键词随机组合 |
| 批量图片编辑 | 调用图片编辑器批量处理 |
| 批量SKU修改 | 表格内直接编辑 |
| 批量价格调整 | 公式计算（如统一加价10%） |
| 批量类目设置 | 类目选择器 |
| Excel粘贴支持 | 产品标题搜索支持逗号分隔和Excel复制粘贴 |

### 4.6 平台适配器设计

#### 4.6.1 字段映射策略

```typescript
interface PlatformAdapter {
  // 字段映射
  mapFieldsToPlatform(product: UnifiedProduct): PlatformSpecificFields;
  
  // 平台特有字段
  getPlatformExtensions(): PlatformExtensionSchema;
  
  // 验证规则
  validate(product: UnifiedProduct): ValidationResult;
  
  // 提交到平台
  submitToPlatform(data: PlatformSpecificFields): Promise<SubmitResult>;
  
  // 同步平台变更回CBHunter
  syncFromPlatform(platformData: any): UnifiedProduct;
}
```

#### 4.6.2 各平台适配器重点

| 平台 | 适配器重点 |
|------|-----------|
| Shopee | SSP/Hot Listing字段、Smart Diagnosis、Advance Fulfillment库存 |
| TikTok | SPS评分关联、CRM配额、跨境/本地店权限区分 |
| TEMU | SPU/SKC/SKU三层、供货价模式、合规强制要求、备货仓组 |
| 妙手 | 采集箱集成、货源价格分离、批量操作模板 |

### 4.7 开发优先级建议

#### P0（第一期）
1. 统一商品数据模型（Product + SKU Table + Platform Extension）
2. 基础编辑弹窗（基本信息 + 图片上传 + 类目选择）
3. SKU管理（简单模式：规格一/规格二）
4. 平台适配器框架（字段映射 + 提交）

#### P1（第二期）
1. 动态表单引擎（类目属性Schema驱动）
2. 批量编辑功能（标题/图片/SKU/价格）
3. 图片编辑器集成（基础版）
4. TEMU三层SKU模式（SPU/SKC/SKU）

#### P2（第三期）
1. 易可图式增强图片编辑器
2. AI标题优化/关键词组合
3. 平台特有功能集成（Shopee Hot Listing、TikTok CRM、TEMU合规）
4. Excel批量导入/导出

## 五、关键设计决策总结

| 决策点 | 推荐方案 | 理由 |
|--------|----------|------|
| 编辑形式 | 弹窗式（妙手风格） | 批量操作友好，上下文保持 |
| SKU管理 | 两层+三层混合 | 兼容普通平台和TEMU全托管 |
| 图片编辑 | 组件化iframe嵌入 | 解耦，可替换 |
| 动态表单 | Schema驱动 | 类目属性动态加载 |
| 字段存储 | JSONB扩展 | 平台差异大，固定表难以覆盖 |
| 定价引擎 | 统一CNY存储 + 国别格式化 | Temu供货价/Shopee零售价都能表达 |
| 批量编辑 | 弹窗内表格编辑 | 参考妙手ERP效率最高 |
