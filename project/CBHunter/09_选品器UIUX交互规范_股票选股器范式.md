### 2.2 策略卡片样式

每个原生策略以卡片形式展示：

```
┌─────────────────────────────────┐
│ 🔥 BS-01 平台榜单Top N    [A级] │
│                                 │
│ 适配: TikTok Shopee Temu        │
│ 状态: ✅ 数据可用                │
│                                 │
│ 参数: TopN=50, 周期=7d          │
│                                 │
│ [添加到工作区] [查看详情]        │
└─────────────────────────────────┘
```

**颜色编码**：
| 数据级别 | 标签颜色 | 说明 |
|----------|----------|------|
| A级 | 🟢 绿色 | 公域数据可直接获取 |
| B级 | 🔵 蓝色 | 需店铺授权 |
| C级 | 🟡 黄色 | 需人工导入 |
| D级 | ⚪ 灰色 | 无数据源，预留 |

### 2.3 拖拽交互

- **拖拽策略到工作区**：从左侧策略库拖入中央条件组合区
- **拖拽调整顺序**：在工作区内拖拽改变策略优先级
- **拖拽删除**：拖出工作区即移除
- **双击策略**：打开参数配置抽屉

## 三、中央条件组合工作区

### 3.1 策略节点可视化

每个策略以"条件卡片"形式展示，节点之间用连线表示运算关系：

```
  ┌──────────────┐     ∩      ┌──────────────┐
  │ BS-01 榜单   │ ─────────→ │ BL-01 蓝海    │
  │ TopN=50      │            │ 搜索量>1000   │
  │ 周期=7d      │            │ 商品数<100    │
  └──────────────┘            └──────────────┘
        │                           │
        │                           │
        ▼                           ▼
  ┌──────────────┐     ∪      ┌──────────────┐
  │ PF-01 利润   │ ←────────── │ RC-01 合规   │
  │ 毛利率≥30%   │            │ 禁限售规避   │
  └──────────────┘            └──────────────┘
        │
        ▼
  ┌──────────────────────────────────┐
  │  运算结果: 42个候选商品           │
  │  [查看商品列表] [保存到策略]      │
  └──────────────────────────────────┘
```

### 3.2 运算符号切换

在两个策略节点之间的连线上，可点击切换运算类型：

```
[∩ 交集] → [∪ 并集] → [− 差集] → [Σ 加权]
```

**运算语义**：
| 运算 | 图标 | 含义 | 示例 |
|------|------|------|------|
| 交集 | ∩ | 同时满足 | 爆款 AND 蓝海 |
| 并集 | ∪ | 满足任一 | 爆款 OR 趋势上升 |
| 差集 | − | 排除 | 爆款 MINUS 高竞争 |
| 加权 | Σ | 综合评分 | 0.4×爆款 + 0.3×利润 + 0.3×趋势 |

## 五、核心交互流程

### 5.1 新建选品任务

```
1. 点击顶部「新建策略」
   ↓
2. 从左侧策略库拖入3-5个原生策略到工作区
   ↓
3. 用连线连接策略节点，选择运算类型（∩/∪/−/Σ）
   ↓
4. 点击各策略的⚙️配置参数和阈值
   ↓
5. 点击「运行选品」
   ↓
6. 查看结果概览 + 商品列表
   ↓
7. 勾选感兴趣的标杆商品
   ↓
8. 执行联动操作：
   ├─ 加入素材库
   ├─ 创建商品草稿
   ├─ 设置SKU定价
   └─ 批量上架到目标店铺
```

### 5.2 保存与复用策略

```
方式1: 保存当前组合
  → 输入策略名称
  → 选择分组
  → 保存为复合策略

方式2: 另存为全新策略
  → 基于当前组合生成新策略ID
  → 可作为子节点继续嵌套

方式3: 导出为模板
  → 分享给团队成员
  → 或发布到公共模板库
```

### 5.3 策略迭代

```
已保存策略 CS-001 v1.0
  → 点击「编辑」进入工作区
  → 调整参数/增减节点/切换运算
  → 点击「保存新版本」
  → 生成 CS-001 v2.0
  → 旧版本保留，可回溯对比
```

## 七、响应式与无障碍

### 7.1 响应式断点

| 断点 | 布局调整 |
|------|----------|
| ≥1440px | 完整三栏布局（策略库 | 工作区 | 结果） |
| 1024-1439px | 策略库可折叠，工作区+结果并排 |
| 768-1023px | 策略库为侧滑抽屉，工作区全屏 |
| <768px | 单栏流程式，逐步引导用户构建策略 |

### 7.2 无障碍要求

- 所有策略节点支持键盘聚焦和Enter激活
- 运算符号切换支持Tab导航
- 颜色编码不单独作为信息载体（配合图标/文字）
- 结果图表提供数据表格备选视图

## 九、v2 现代化动态智能选品交互（新增）

### 9.1 交互范式升级总览

v1采用传统"分类浏览→勾选→筛选"模式。v2升级为**量化智能选品操作范式**，对标TradingView策略测试器、同花顺iFinD智能选股、QuantConnect量化平台：

```
v1: 分类浏览 → 手动勾选 → 静态筛选 → 一次性运行
v2: AI生成建议 → 可视化拖拽搭建 → 实时参数预览 → 动态权重调配 → 智能负向排除 → 一键衍生迭代
```
